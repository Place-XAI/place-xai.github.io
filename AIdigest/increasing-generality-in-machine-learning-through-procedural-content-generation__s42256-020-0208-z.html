Increasing generality in machine learning through procedural content generation

Sebastian Risi · Julian Togelius Published 2020-08-03 DOI 10.1038/s42256-020-0208-z Source nature.com ↗
Core Conclusion

Procedural content generation (PCG) — the algorithmic creation of game levels, maps, characters, and rules — is not merely a game-design technique: it is a systematic toolbox for making ML systems more general. By exposing training agents to algorithmically varied environments (domain randomization, curriculum generation, co-evolution), PCG counteracts overfitting to fixed training scenarios, enables robust sim-to-real transfer (e.g., a robot hand solving a Rubik's cube trained entirely in simulation), and provides a principled path toward open-ended, lifelong machine intelligence.

  • Names the core failure mode of modern ML. Deep RL agents overfit to fixed training environments — training on a single set of Atari levels produces brittle policies that fail when any parameter changes. PCG is reframed as the principled antidote to this task-level overfitting.

  • Demonstrates high-profile real-world payoff. The OpenAI Rubik's cube robot hand — trained only in simulation with expanding domain randomization — shows that PCG-based training already enables physical robots to operate in the real world without additional training.

  • Reframes data augmentation as PCG. ImageNet augmentation, domain randomization, and AutoAugment are all instances of constructive PCG — unifying disparate literatures under a single conceptual umbrella with decades of richer prior art.

Core Methods

  • Constructive PCG: fixed-time, non-iterative generation without search — fractal noise (Perlin), L-systems, cellular automata; the simplest and fastest category, conceptually equivalent to image data augmentation (cropping, padding, noise, stylization).
  • Search-based PCG: stochastic optimization (evolutionary algorithms, CMA-ES, differential evolution, Map-Elites) over a content space, guided by an evaluation function — generates game levels (Mario, Doom, Angry Birds), board games (Yavalath), and diverse artifact populations.
  • PCG via ML: generative models (GANs, LSTMs, Markov chains) trained on existing content; latent variable evolution searches the GAN latent space; RL-based PCG treats content generation as a sequential decision process rewarded by content quality.
  • Domain randomization (uniform / guided / automatic): training across simulated environments with randomized parameters (textures, masses, lighting, geometry) to create policies robust to distribution shift; automatic domain randomization scales parameter diversity as agent performance improves.
  • Procedurally generated learning environments / curriculum: co-evolutionary methods (MCC, POET) simultaneously evolve environments and agents; Progressive PCG adjusts level difficulty based on agent performance — enables curriculum learning and promotes meta-learning emergence in RNNs.
  • Bootstrapping for functional content: only newly generated artefacts passing a verifiable functionality check (e.g., A*-confirmed level playability) are added back to the training set, iteratively biasing generation toward valid functional content despite limited initial data.

② Use POET-style co-evolutionary curriculum for facility-location optimization across city morphologies

Extending this paperThe paper's POET and PPCG frameworks generate progressively harder environments matched to agent capability — the key insight being that a fixed-difficulty environment either frustrates or bores a learning agent, and that a fixed set of training environments overfits the agent to those specific configurations. A direct paper extension: apply co-evolutionary curriculum to the facility-location optimization problem, where an RL optimizer (agent) places services to maximize spatial accessibility. Instead of presenting all city topologies at once, PCG would generate progressively more complex city morphologies — from regular grids to polycentric, dendritic, and fragmented networks — so the optimizer generalizes across morphologies never seen during training, exactly the challenge the paper proposes for "PCG beyond game-like environments."

A curriculum-guided RL agent trained with PCG-varied city topologies would learn a transferable placement policy — the key step toward a GeoAI facility-location solver that generalizes across cities without per-city re-optimization. This directly extends my facility-location and urban accessibility research lines, and aligns with the physics-informed graph learning direction where city topologies are encoded as graph structures.

③ Bootstrapped functional-content generation for verified synthetic spatiotemporal scenarios

Extending this paperThe paper highlights that GAN-based PCG generates content that "looks right but doesn't function" — a Super Mario Bros level with an impossible jump is unplayable and worthless. Its solution, bootstrapping, verifies generated content (e.g., A* confirms level traversability) and adds only verified-functional artefacts back to the training set. This exact problem recurs in synthetic spatiotemporal data generation: a generated mobility trajectory or epidemic-spread scenario may look statistically plausible but violate geographic constraints (impossible road traversal times, population flows through closed borders, negative density values). Extending the paper: define a geographic functionality oracle — a constraint checker analogous to A*-based level verification — and bootstrap synthetic scenario generation so only geographically valid synthetic data enters the training corpus for spatiotemporal ML.

Added value to my researchMy work fuses mobile-phone GPS trajectories and social media data to model spatiotemporal phenomena (SIR epidemics, job-housing flows, health service utilization). Bootstrapped PCG for synthetic spatiotemporal data would let me train models on a much larger verified corpus, directly addressing data scarcity for rare-event scenarios (disease outbreaks, extreme commuting disruptions, post-disaster mobility). This supports both my data-science/reproducibility agenda and the need for robust, geographically transferable spatiotemporal models.

④ Close the digital-twin gap in urban health modeling via automatic domain randomization

Extending this paperThe paper discusses the content gap in sim-to-real transfer: even excellent domain randomization fails when real environments contain scenarios outside the simulation's PCG distribution. The solution it proposes — automatic domain randomization that progressively expands parameter diversity as the agent improves — is also the prescription for urban modeling. A direct extension: apply automatic domain randomization to urban digital twins, progressively broadening the synthetic distribution of city parameters (population sparsity, road density, facility count, socioeconomic gradient) during training, so a model covers the tail of the real-world distribution and does not fail in underrepresented city types. This is precisely "from simulation to the real world" applied outside robotics.

Added value to my researchMy health geography work — modeling spatial accessibility of primary care and telehealth across U.S. counties — faces exactly this content-gap challenge: models calibrated on dense urban counties fail systematically in rural frontier areas. Automatic domain randomization over synthetic county configurations (population sparsity, road network density, facility ratios) would harden my 2SFCA/R2SFCA accessibility models against distributional shift without requiring expensive rural data collection campaigns, directly improving their policy utility for underserved populations.

⑤ Build a PCG-based incremental benchmark for geographic generalization of spatial ML models

Extending this paperOne of the paper's six open challenges is creating PCG-based benchmarks in which environments become progressively harder, revealing whether an RL agent truly generalizes or merely memorizes — a benchmark property the existing Atari suite entirely lacks. The analogous gap exists in GeoAI: there is no agreed benchmark that tests whether a spatial ML model generalizes across increasing levels of geographic heterogeneity (mono-centric to polycentric, dense to sparse, homogeneous to mixed land-use). Extending the paper: design a PCG-generated suite of synthetic cities spanning a controlled difficulty axis (urban morphology complexity, data density, population heterogeneity) and release it as an open geographic generalization benchmark — the GeoAI equivalent of CoinRun or GVG-AI for the geospatial ML community.

Added value to my researchMy lab's open-science and FAIR data directions position me to lead exactly this kind of benchmark contribution. A PCG-generated geographic generalization benchmark would provide a reproducible evaluation scaffold for explainable spatial ML prediction, MGWR/GWR non-stationarity testing, and physics-informed graph learning for urban networks — giving the GeoAI community a common generalization axis analogous to what the arcade learning environment provided to RL research, and directly supporting my research-reproducibility agenda.

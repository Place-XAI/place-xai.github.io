A software framework for end-to-end genomic sequence analysis with deep learning

Adam Klie · Hannah Carter Published 2023-11-16 DOI 10.1038/s43588-023-00557-5 Source nature.com ↗
Core Conclusion

Deep learning is increasingly powerful for gene regulation research, but the field lacks a cohesive, reusable toolchain. EUGENe (Elucidating the Utility of Genomic Elements with Neural Nets) fills this gap as a FAIR, modular, end-to-end Python toolkit that unifies data extraction, transformation, GPU-accelerated model training, and sequence-feature interpretation for regulatory genomics — lowering the barrier for both novice and expert users and advancing interoperability and reproducibility in the field.

Why It Matters

  • Closes a critical software gap. Genomics deep learning has been held back by fragmented, model-specific codebases; EUGENe provides a single FAIR framework covering the full workflow from raw sequence files to model interpretation.
  • Democratizes cutting-edge sequence modeling. The toolkit is designed to be usable by researchers with only basic programming skills, widening access to state-of-the-art methods such as sequence convolutional networks and attention models.
  • Enables large-scale analysis without memory bottlenecks. EUGENe trains on datasets too large to fit in CPU memory by using Zarr/Dask lazy loading, a feature currently unavailable in competing tools.
  • Plug-and-play benchmarking ecosystem. Integration with the Kipoi model zoo lets users instantiate and compare dozens of published genomics models in a uniform API — accelerating reproducibility and benchmarking.
  • Seeds a developer community. Following the scverse blueprint, EUGENe is positioned to become a community-maintained platform for regulatory genomics, not just a one-off publication code release.

Core Methods

  • Extract–Transform–Load (ETL) pipeline: SeqData module parses genomic file formats (FASTA, BED, VCF, BigWig), converts them to compressed Zarr stores, and lazy-loads them via Dask/XArray — decoupling data volume from CPU memory.
  • Sequence preprocessing: SeqPro module handles normalization, one-hot encoding, data augmentation, and other transformations needed to prepare raw sequences for model input.
  • Model instantiation: EUGENe API supports three model-building pathways: (1) YAML configuration files, (2) programmatic API calls using EUGENe's building-block library (CNN layers, attention heads, hybrid architectures), and (3) direct import from the Kipoi model zoo.
  • GPU-accelerated training: PyTorch Lightning backend handles distributed/single-GPU training with minimal boilerplate, including callbacks for logging, early stopping, and checkpointing.
  • Feature attribution / interpretation: SeqExplainer module applies Captum-backed methods (gradient saliency, integrated gradients, DeepLIFT) to trained models and renders results as nucleotide-level sequence-logo visualizations identifying transcription-factor binding sites (TFBS).
  • Modular architecture: Every step (data, model, training, interpretation) is a replaceable module — users can substitute their own data loaders, custom model classes, or external attribution methods without rewriting the pipeline.

① Extend EUGENe's SeqExplainer with spatial-aware attribution for genomic loci

Extending this paperEUGENe's current SeqExplainer treats every nucleotide position equivalently using standard gradient-based attribution (integrated gradients, DeepLIFT). However, regulatory elements are spatially clustered along chromosomes, and attribution maps inherit this genomic positional structure. A direct extension: introduce a spatially-aware attribution smoothing layer — analogous to spatially-weighted regression — that down-weights isolated attribution spikes and amplifies contiguous motif clusters. This would produce biologically more interpretable sequence logos and reduce false-positive motif calls in SeqExplainer, making EUGENe's interpretation outputs more reliable for identifying TFBS.

Added value to my researchThis is structurally identical to my work on spatially-varying coefficient models (MGWR/GWR): in both cases, we want to discover where a signal is locally elevated rather than averaging it globally. The design pattern — spatially-weighted attribution with adaptive bandwidth — could be directly ported from geospatial kernel methods to sequence-position kernels. This cross-domain transfer would enrich my explainable ML toolkit with a new interpretability lens applicable wherever positional context matters (genetic loci, street segments, spatial grids).

② Build a FAIR federated learning layer on top of EUGENe for multi-institutional genomics

Extending this paperEUGENe currently assumes all genomic data can be centralized in one compute environment. Many clinical and population genomics datasets cannot be shared due to privacy constraints. A natural extension of the paper's FAIR-software mandate: add a federated training module so multiple institutions can contribute gradient updates from local sequence data without pooling raw genomes. The Zarr-based ETL is already partition-friendly; adding federated aggregation (FedAvg or differential-privacy wrappers) would make EUGENe the first end-to-end federated deep learning framework for regulatory genomics and dramatically widen its data scope beyond what any single lab can access.

Foundation models pre-trained on entire genome assemblies (e.g., Enformer, Nucleotide Transformer, HyenaDNA) now offer far richer sequence representations than random initialization. This would position EUGENe as the fine-tuning harness for the next generation of genomics foundation models, directly applying the paper's FAIR-software and end-to-end principles to large pre-trained systems.

⑤ Leverage EUGENe's training-data generalization lessons for geospatial model evaluation

Extending this paperEUGENe explicitly documents "machine learning pitfalls" and encourages users to read method-specific failure-mode documentation. A critical pitfall for regulatory genomics — and flagged in the companion corpus — is that model generalization depends heavily on training data composition (positive vs. negative example balance, genomic context diversity). EUGENe could be extended with an automated data-composition auditor: before training, scan the SeqData object for class imbalance, sequence redundancy, and chromatin-context coverage gaps, and report a data-quality scorecard. This would make EUGENe's FAIR promise operational rather than aspirational.

Added value to my researchTraining-data composition is equally critical in my spatial ML workflows: healthcare access models trained on urban cores fail in rural areas, and mobility models trained on weekdays fail on weekends. Formalizing an analogous spatial data-composition audit — checking for geographic coverage gaps, demographic imbalance, and temporal distribution shifts before model fitting — would strengthen the reproducibility of my GeoAI pipelines and align with my open science / FAIR data principles.

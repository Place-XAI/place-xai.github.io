Algorithmic fairness and bias mitigation for clinical machine learning with deep reinforcement learning

Jenny Yang · Andrew A. S. Soltan · David W. Eyre · David A. Clifton Published 2023-07-31 DOI 10.1038/s42256-023-00697-3 Source nature.com ↗
Core Conclusion

A deep reinforcement learning (RL) framework with a specialized, fairness-aware reward function can mitigate both ethnicity-based and hospital site-specific biases in clinical machine learning, while maintaining clinically effective screening performance for COVID-19. Evaluated across four UK NHS Trusts (over 77,000 patient presentations in total), the RL approach achieves the best or second-best equalized-odds outcomes compared with adversarial debiasing, XGBoost, and neural network baselines — and generalizes to an independent ICU discharge-prediction task using the publicly available eICU database.

★

Why It Matters

  • Tackles a clinically urgent equity problem. ML models deployed in healthcare can amplify existing racial and institutional disparities; this work provides a concrete, deployable mitigation strategy grounded in real NHS emergency-department data.
  • Novel framing of fairness as a reward signal. Rather than adversarial debiasing or post-hoc reweighting, the paper encodes fairness directly into the RL reward — enabling it to control when and how error signals propagate, bypassing cross-entropy aggregation that favors majority groups.
  • Handles multi-class sensitive attributes. The reward function is formulated to debias categorical attributes with more than two classes (e.g., multiple ethnic groups), which is clinically more realistic than binary-only approaches.
  • External multi-site validation. Models trained at Oxford University Hospitals (OUH) are externally validated at three independent NHS trusts (PUH, UHB, BH) and tested on the publicly available eICU database — demonstrating true generalizability, not just held-out performance on one hospital's data.
  • Reproducible and open. Code is released on Code Ocean and GitHub, enabling the community to audit the debiasing procedure and adapt it to other clinical screening tasks.

Core Methods

  • Duelling Double Deep Q-Network (DDQN): classification is reframed as a finite Markov decision process; the agent selects COVID-19/negative actions given EHR features as states. A duelling architecture separately estimates state value V(s) and action advantages A(s,a), reducing overoptimistic Q-value estimates via the double Q-learning update.
  • Fairness-aware reward function: the reward is split into a classification component (negative reward inversely proportional to class frequency) and a debiasing component (positive reward inversely proportional to the minority sensitive-attribute class frequency). This makes the agent more sensitive to under-represented ethnic groups or smaller hospital cohorts without relying on gradient-aggregating cross-entropy loss.
  • Multi-class sensitive attribute debiasing: the reward function is formulated for arbitrary K classes of the sensitive attribute z, enabling debias of multi-category ethnicity labels (not just binary).
  • ε-greedy training with episode termination: a terminal state is triggered when the agent misclassifies a minority-class sample, creating a dependency between state and action that RL exploits to focus learning on hard, underrepresented cases.
  • Comparators: standard RL (no debiasing), XGBoost, fully connected neural network (both with and without cost-adjusted weights), and adversarial debiasing — all with identical feature sets and threshold-optimization procedures.
  • Threshold adjustment: decision boundary optimized via grid search to achieve sensitivity ≥ 0.9 on the validation set (exceeding lateral flow device sensitivity of 56.9% and matching PCR sensitivity of ~80–90%).
  • Evaluation: AUROC, sensitivity, specificity, PPV, NPV (95% CI by Hanley–McNeil); fairness by equalized odds quantified as standard deviation of true positive rates and false positive rates across sensitive-attribute classes (lower = fairer); statistical significance via Wilcoxon signed rank test (P < 0.0001).
  • t-SNE visualization: used to confirm site-specific distributional shifts in the COVID-19 positive training data, motivating the hospital-bias mitigation task.

① Spatialize the reward function: geography as a sensitive attribute

Extending this paperThe paper treats hospital site and ethnicity as discrete sensitive attributes, but both are proxies for spatial disadvantage — patients from deprived neighborhoods, rural postcodes, or spatially under-resourced catchments. A direct extension of the paper's DDQN framework would encode geographic deprivation indices or travel-time to care as a continuous sensitive attribute z, converting the reward function's per-class weights into a spatially-weighted signal. This would allow the model to mitigate biases that arise from where patients live, not just who they are — a dimension the current paper explicitly acknowledges as a gap (ethnicity "encompasses place of residence and socioeconomic status") but does not operationalize.

Added value to my researchThis is a natural coupling of the paper's RL debiasing framework with my 2SFCA/R2SFCA accessibility modelling: the catchment-area travel-time scores I compute for healthcare facilities can directly enter the reward function as the continuous spatial sensitive attribute, creating a debiased clinical prediction model that is simultaneously aware of access inequality. It also opens a pathway to spatially explicit fairness metrics — a gap in both the healthcare ML and GeoAI literatures.

② Replace static equalized odds with a dynamic, distribution-shift-aware fairness criterion

Extending this paperThe paper evaluates fairness via equalized odds computed on fixed train/test splits, but the t-SNE visualization (Fig. 3) itself shows that distributional shifts between sites are real and ongoing — thresholds calibrated for OUH may be miscalibrated for a new hospital. Extending the paper: pair the RL agent with a causal model of measurement bias (admission bias, sampling bias, observer bias — all named in the introduction) to learn a reward function that remains fair under covariate shift. A causal framing also answers the harder question the paper raises in Discussion: whether observed ethnicity differences in COVID-19 risk reflect true biological signal or data-collection artifacts.

Added value to my researchThis bridges to my spatiotemporal analysis agenda: causal inference about distributional shifts across space and time is precisely what my spatiotemporal kernel density estimation and GWR methods aim at. A causal debiasing layer for EHR data would generalize my existing work on spatial non-stationarity in health outcomes into the clinical prediction domain, making my methods applicable to NHS-scale datasets.

③ Add explainability to the fairness: which features drive bias and which drive accuracy?

Extending this paperThe paper demonstrates that the RL model is fairer, but does not explain which input features the model relies on differently across ethnic groups or hospital sites. Extending it: apply post-hoc attribution (e.g., SHAP or integrated gradients) stratified by sensitive attribute z to decompose the Q-value into feature-level fairness and accuracy contributions. This would operationalize the paper's Discussion call to "understand the complex interplay between genetic, social, and behavioural factors in clinical outcomes" and address the interpretability requirements of NHS clinicians — who, as a contemporaneous NMI paper notes, have diverse and task-specific interpretability needs.

This is directly applicable to my telehealth and mental health accessibility work, where model trust among underserved populations is a first-order concern.

④ Embed the debiasing reward in a federated multi-hospital learning setup

Extending this paperThe paper's training strategy pools data from multiple NHS Trusts for the hospital-bias task, but notes that models can still acquire site-specific biases during training due to varying data volumes — OUH contributes far more data than BH. Extending it: implement the DDQN debiasing framework within a federated learning setup where each hospital trains its own Q-network locally, and a central coordinator aggregates models (FedAvg or FedProx). The fairness reward function would then operate at the site level — each hospital's local agent is simultaneously debiased for its own demographic distribution, while the global model benefits from cross-site generalization without raw data sharing, addressing the HRA privacy requirements the paper cites.

⑤ Apply the fairness-reward paradigm to spatial facility-allocation decisions

Extending this paperThe paper's core innovation — a reward function that is inversely proportional to the minority group's representation — is domain-agnostic. The Discussion explicitly states: "Although we use clinical case studies, the framework introduced can be generalized across many different domains." Extending it to facility-location optimization: treat each candidate facility site as an action, the spatial distribution of underserved populations as the sensitive attribute z, and the coverage score as the base reward. The RL agent would then learn a facility-placement policy that is simultaneously efficient (maximizes total coverage) and equitable (reduces coverage disparity by neighborhood deprivation or ethnicity), with the fairness reward automatically up-weighting placements that serve minority catchments — directly operationalizing equalized odds in a spatial planning context.

Added value to my researchThis is the most direct bridge to my EV charging-station efficiency-versus-equity optimization and healthcare facility access work. The current RL framework I know from the literature typically optimizes coverage or travel time; inserting the paper's fairness reward as a second objective would give my facility siting models an explicit, theoretically grounded equity component anchored in equalized-odds rather than ad hoc weighting — a publishable methodological contribution in GeoAI and spatial planning.

Controlling diverse robots by inferring Jacobian fields with deep networks

Sizhe Lester Li · Annan Zhang · Boyuan Chen · Hanna Matusik · Chao Liu · Daniela Rus · Vincent Sitzmann Published 2025-06-25 DOI 10.1038/s41586-025-09170-0 Source nature.com ↗
Core Conclusion

Deep neural networks can learn a visuomotor Jacobian field — a dense 3D map of how every point on a robot's body responds to actuator commands — from a single camera, trained entirely self-supervised on random-command videos. The method makes no assumptions about materials, actuation or embedded sensing, and achieves accurate closed-loop control across a diverse set of robots including soft pneumatic hands, handed-shearing-auxetic platforms, a 16-DoF rigid hand, and a low-cost 3D-printed arm. By decoupling hardware design from manual modelling expertise, the approach substantially broadens the design space of controllable robots and lowers the barrier to robotic automation.

Core Data — Sources & Volume

Training data
Self-collected multi-view video datasets for each robot platform: 2–3 hours of video per robot, captured by 12 Intel RealSense D415 RGB-D cameras observing random-command executions. Publicly available at HuggingFace.
Robot platforms
Four physically diverse manipulation systems: (1) 3D-printed hybrid soft–rigid pneumatic hand (15 pneumatic channels, ~US$300); (2) Handed Shearing Auxetics (HSA) parallel manipulator (4 DoF); (3) Allegro Hand (16 DoF, commercial); (4) Poppy Ergo Jr 6-DoF arm (open-source, low-cost, substantial backlash).
Data collection
Random command sampling from a safe per-channel range; each data sample captures a steady-state image tuple before and after a commanded motor delta across all 12 cameras. No human annotation required.
Optical flow
Off-the-shelf RAFT optical flow computed between consecutive multi-view frames; TAPIR used for 2D point tracking at test time; segmentation masks from Segment Anything.
Geometry supervision
RGB-D depth from Intel RealSense cameras; camera poses calibrated with 3-cm AprilTags. Depth loss is optional — pure RGB training is also feasible.

Core Methods

  • Visuomotor Jacobian field: a deep neural network maps a single RGB image I to a 3D spatial field J(x, I) = ∂x/∂u, encoding how every 3D point x moves under actuator command change δu. Generalizes the classical system Jacobian to a continuous, vision-conditioned spatial representation.
  • Neural radiance field (NeRF) backbone: both the Jacobian field and a radiance field (geometry + appearance) are predicted jointly from one image using the pixelNeRF architecture — bilinear-interpolated 2D feature volumes projected to 3D via sinusoidal positional encoding.
  • Self-supervised training via differentiable rendering: predicted 3D motion fields are volume-rendered to optical flow and compared with RAFT-computed flows; radiance field is rendered to RGB-D and compared with camera captures. No human labels needed.
  • Model predictive control (MPC): at inference, the Jacobian field is queried to solve for the actuator command minimizing L2 distance to 2D pixel-space waypoints (or Wasserstein-1 distance for 3D point-cloud tracking). Runs at approximately 12 Hz.
  • Inductive biases: the Jacobian parameterization enforces linearity, spatial locality and compositionality — properties of real mechanical systems — which are shown to improve generalization over direct scene-flow prediction baselines.
  • Domain randomization: robot foregrounds are overlaid on random natural-scene backgrounds during training (using SAM masks) to improve robustness to occlusion and background change at deployment.

① Transfer the Jacobian field's self-supervised 3D field representation to spatiotemporal urban sensing

Extending this paperThe paper learns a dense 3D sensitivity field entirely from raw video and random commands, with no labeled data. This self-supervised regime is powerful because it encodes causal structure (which actuator moves which part) without annotation. A direct extension would replace the robot's actuator commands with policy interventions — for example, transit schedule changes — and define the "Jacobian" as the sensitivity of urban flow fields to those interventions, training from GPS or mobile-phone mobility streams. The paper's volume-rendering supervision could be replaced by spatiotemporal kernel density rendering of observed mobility trajectories.

② Extend the Jacobian's "physics without a model" concept to physics-informed graph learning for urban networks

Extending this paperThe paper's core insight is that the linearization J(x, I) = ∂x/∂u — the local sensitivity of any 3D point to any actuator — can be learned directly from observations, bypassing the need to solve the governing PDE (continuum mechanics of soft materials). The same idea applies to urban network dynamics: rather than solving a traffic flow PDE or an epidemic SIR system from scratch, one could learn a network Jacobian field that maps node-level interventions to system-wide state changes. The paper's differentiable-rendering training loop directly inspires a differentiable network simulation layer as training supervision.

Added value to my researchMy physics-informed graph learning agenda seeks to encode conservation and flow priors into GNNs for urban systems. The Jacobian-field formulation offers a concrete architecture: encode the network graph as the "camera" projection (the spatial prior), learn the Jacobian of node-level outcomes (disease incidence, transit ridership) with respect to facility placements, and supervise using observed spatiotemporal outcomes via differentiable aggregation — a physics-inspired, annotation-free GNN for health geography.

③ Use the paper's causal structure discovery to build explainable ML for spatial prediction

Extending this paperThe paper demonstrates a striking property: the Jacobian field automatically discovers which command channels actuate which parts of the robot, without any supervision (Fig. 2 colour coding). Extending the paper: apply the same architecture to multi-channel interventional data in biology or social systems, where "actuators" are policy levers and "robot geometry" is the state of a spatial system.

④ Apply the paper's generalization across robot morphologies to foundation-model transfer for geospatial domains

Extending this paperA key result is that a single algorithm — with no per-robot specialization — generalizes across radically different morphologies (soft pneumatic, auxetic, rigid, backlash-prone). Extending this: train a single Jacobian-field-style model across a large library of robot types, and study whether a foundation Jacobian model emerges that zero-shots new robot morphologies — the robot analogy of a geospatial foundation model generalizing to unseen city types.

⑤ Adopt the paper's MPC-over-Jacobian-field paradigm for real-time healthcare facility access optimization

Extending this paperThe paper's closed-loop controller works by: (1) query the Jacobian field to predict how a command changes the state; (2) solve for the command that minimizes distance to the goal waypoint; (3) execute and observe. This predict-and-optimize loop runs at 12 Hz for robots. Extending the paper: replace "robot command" with "facility opening/closure decision," replace "Jacobian field" with a learned accessibility sensitivity field, and replace "waypoint" with a target equity-accessibility distribution. The result is a real-time, data-driven facility-location optimizer that closes the loop on observed population outcomes.

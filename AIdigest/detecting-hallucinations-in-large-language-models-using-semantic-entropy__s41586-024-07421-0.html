Detecting hallucinations in large language models using semantic entropy

Sebastian Farquhar · Jannik Kossen · Lorenz Kuhn · Yarin Gal Published 2024-06-19 DOI 10.1038/s41586-024-07421-0 Source nature.com ↗
Core Conclusion

Hallucinations (specifically confabulations — arbitrary, fluent falsehoods) in LLMs can be detected without domain knowledge by measuring semantic entropy: entropy computed over clusters of meaning rather than over raw token sequences. Semantic entropy achieves an average AUROC of 0.790 across 30 model–dataset combinations, outperforming supervised and naive entropy baselines, works for sentence-length QA (TriviaQA, SQuAD, BioASQ, NQ-Open, SVAMP) and paragraph-length biography generation (FactualBio/GPT-4), and generalizes out-of-distribution without any labelled examples.

Why It Matters

  • Closes a critical safety gap in LLM deployment. Users of ChatGPT, Gemini, and similar systems cannot currently tell which outputs to trust. Semantic entropy provides a model-agnostic, task-agnostic signal that flags when the model is likely to confabulate — enabling selective abstention or human review.
  • Principle over heuristic. Rather than curating failure examples, the method derives uncertainty from first principles in probabilistic machine learning — computing entropy in the space of meanings rather than tokens — making it applicable to any generative model.
  • Demonstrates confabulations are the dominant error mode. Semantic entropy outperforms even supervised classifiers trained on the same distribution, implying that arbitrary incorrect generations (confabulations) are a larger error category than systematic biases from training data.
  • Enables a discrete black-box variant. By approximating probability clusters from generation counts alone, the method works without access to output token probabilities — extending applicability to proprietary APIs like GPT-4.
  • High-stakes domains. Confabulations in medical (radiology), legal (precedents), and journalistic contexts are not hypothetical risks; this paper frames a detection method with direct application to safety-critical deployments.

Core Methods

  • Semantic entropy (SE): sample M outputs from the LLM given a prompt; cluster outputs by bidirectional entailment (if A entails B and B entails A, they share a meaning-cluster); compute Shannon entropy over cluster probabilities (summing token probabilities within each cluster). High SE = likely confabulation.
  • Discrete semantic entropy: approximates cluster probabilities from generation counts alone (no token probabilities required) using the empirical categorical distribution — enabling application to black-box APIs such as GPT-4.
  • Bidirectional entailment clustering: uses DeBERTa-Large-MNLI or GPT-3.5 to check whether two candidate answers logically entail each other in the context of the question, building semantic equivalence classes transitively.
  • Factoid-decomposition pipeline for long text: (i) decompose paragraph into factual claims; (ii) generate Q questions per factoid; (iii) resample M answers per question; (iv) compute SE per question; (v) average over questions as a factoid-level confabulation score.
  • Evaluation metrics: AUROC (measures ability to distinguish correct vs incorrect answers) and AURAC (area under the rejection-accuracy curve — accuracy gain when abstaining on highest-entropy fraction).
  • Baselines: naive length-normalized joint entropy; P(True) prompted LLM self-evaluation (supervised in-context); embedding regression logistic classifier (supervised, trained on hidden states).
  • Rao-Blackwellized Monte Carlo estimation: approximates the full semantic entropy from M=10 samples (sentence-length) or M=3 samples (biography factoids), selected via ablation in supplementary figures.

① Extend semantic entropy from confabulation detection to calibrated confidence for geospatial LLM agents

Extending this paperThe paper demonstrates semantic entropy on general QA benchmarks and biographies, but its authors acknowledge that "algorithmic adaptations to other problems" could unlock new domains. A direct extension: apply semantic entropy to geospatial question answering (e.g., "Which hospitals in rural Kenya have catchment coverage below 30 minutes?") where LLMs are increasingly used as plan-language interfaces over spatial databases. The factoid-decomposition pipeline maps naturally onto multi-hop geospatial reasoning chains, and the discrete variant handles black-box LLM-powered GIS copilots without token-probability access.

Added value to my researchIn my spatiotemporal analysis work, I increasingly use LLMs to synthesize multi-source inputs (mobile phone data, POIs, census) into natural-language policy summaries. Attaching semantic entropy scores to those summaries would allow automatic flagging of unreliable geographic inferences — building a confabulation-aware geospatial AI pipeline that is especially valuable in data-sparse regions where ground truth is unavailable for supervised calibration.

② Use semantic entropy as a curriculum signal for RLHF-free LLM self-improvement

Extending this paperThe paper's method is purely diagnostic: it detects confabulations but does not correct them. The natural next step is closing the loop — using semantic entropy scores to prioritize which generations an LLM should revise or retrieve external knowledge for, without requiring human-labelled preference pairs. This would convert semantic entropy from a passive detector into an active self-calibration signal: high-entropy outputs trigger retrieval-augmented generation (RAG) or chain-of-thought verification while low-entropy outputs are passed through directly, saving compute.

Added value to my researchIn my work fine-tuning geospatial foundation models on multi-source data, training signal quality is the central bottleneck. Using semantic entropy as a curriculum filter — only propagating gradients on low-entropy (confident, consistent) generations — would reduce the noise injected by contradictory spatial descriptions, acting as a self-supervised data quality filter for geospatial instruction tuning.

③ Transfer the bidirectional entailment clustering to spatial-claim consistency checking

Extending this paperThe paper's bidirectional entailment clustering operationalizes semantic equivalence for factual propositions in English. Extending this: adapt entailment clustering to spatial-relational claims (e.g., "Facility A is within 5 km of population centre B" vs "Population centre B is accessible from facility A in under 10 minutes by road") where two phrasings may be spatially equivalent but lexically distinct — requiring a geometry-aware semantic equivalence checker rather than a pure NLI model. The clustering framework stays identical; only the entailment oracle changes.

Added value to my researchThis directly addresses a bottleneck in my 2SFCA/R2SFCA accessibility research: when validating LLM-generated accessibility narratives against GIS computations, I need to determine whether a model's verbal description of a catchment area is factually consistent with the underlying spatial data. A geometry-informed entailment model would allow automated consistency checking between natural-language accessibility reports and GIS outputs.

A direct extension: deploy semantic entropy in a clinical decision-support loop, using the discrete variant (which works without token probabilities, matching the black-box API access typical in health IT), and calibrate AUROC thresholds against expert-annotated clinical question sets. The factoid-decomposition pipeline maps naturally onto multi-sentence clinical reasoning chains.

⑤ Combine semantic entropy with watermarking to build a full provenance-and-reliability stack for LLM outputs

Extending this paperThe paper addresses the question "is this generation likely to be wrong?" but not "was this generation produced by a specific model?" Combining semantic entropy with cryptographic watermarking (as developed in parallel Nature work) would create a two-layer reliability stack: the watermark establishes provenance (which model generated it), while semantic entropy assesses faithfulness (how uncertain that model was). Together they address both fabrication (wrong model) and confabulation (uncertain model) — the two dominant AI-content integrity risks.

Pairing watermarked LLM outputs with semantic entropy confidence metadata would enable reproducible audit trails for AI-assisted geospatial analysis — a concrete contribution to explainable, FAIR-compliant GeoAI pipelines.

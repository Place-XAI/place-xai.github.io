Differentiable visual computing for inverse problems and machine learning

Andrew Spielberg · Fangcheng Zhong · Konstantinos Rematas · Krishna Murthy Jatavallabhula · Cengiz Oztireli · Tzu-Mao Li · Derek Nowrouzezahrai Published 2023-11-17 DOI 10.1038/s42256-023-00743-0 Source nature.com ↗
Core Conclusion

Differentiable visual computing (DVC) is a unifying paradigm that bridges classical 3D computer graphics (geometry, animation, rendering) with modern machine learning by making every stage of the visual computing pipeline end-to-end differentiable. Computing gradients through geometry, physics simulation, and image synthesis enables data-efficient, physics-grounded inverse problem solving — in robotics, manufacturing, autonomous driving, and biomechanics — that pure deep learning cannot match in data efficiency, and that classical forward simulation cannot achieve alone. The field is nascent but rapidly expanding, and a standardized DVC pipeline is both achievable and urgently needed.

★

Why It Matters

  • Closes the gap between simulation and learning. Classical visual computing excels at forward simulation but cannot adapt from data; deep learning is adaptive but ignores physics. DVC merges both — the first systematic framework to do so at pipeline scope.
  • Data efficiency through physical priors. By embedding physics (geometry, dynamics, light transport) as differentiable constraints, DVC drastically reduces the data volume deep learning requires and improves generalization to unseen conditions.
  • Unlocks otherwise intractable inverse problems. Gradient-based optimization through a differentiable pipeline outperforms gradient-free methods (reinforcement learning, evolutionary strategies) by orders of magnitude in robotics, control, and material inference.
  • Covers the breadth of visual computing. The review unifies geometry processing (meshes, SDFs, NeRFs), differentiable physics simulation (rigid, elastic, fluid, cloth), and differentiable rendering (rasterization, ray-tracing, path tracing) into a coherent, jointly differentiable framework.
  • Practical infrastructure path. It identifies concrete tools — PyTorch3D, TensorFlow Graphics, Taichi, Dr. JIT, Halide — that already make DVC deployable, lowering the barrier for practitioners across engineering and science.

Core Methods

  • Automatic differentiation (autodiff / backpropagation): the foundational technique — computing gradients of any pipeline output with respect to any input by composing partial derivatives through a directed acyclic computation graph (DAG). Both forward-mode and reverse-mode (backprop) variants are surveyed.
  • Differentiable geometry processing: strategies for emitting continuous gradients from discrete representations (meshes, point clouds, voxel grids, SDFs, occupancy/density fields, radiance fields). Three approaches: domain smoothing, output smoothing, and neural operators (PointNet, implicit networks, mesh deformation networks).
  • Differentiable physics simulation: constructing differentiable state-transition models S(q, t; θ) for rigid-body, elastic, fluid, and cloth dynamics; gradient propagation via full trajectory rollout + backpropagation; forward-mode differentiation for memory-efficient long-horizon control; implicit function theorem and optimization layers for contact/quasistatic regimes.
  • Differentiable rendering: numerical gradient estimation for image synthesis with respect to geometry (occlusion), materials (reflectance), lighting (multi-scattering), and camera parameters; covers differentiable rasterization, ray-tracing, and path tracing; includes neural scene representations (NeRF) as differentiable rendering components.
  • End-to-end pipeline composition: chaining differentiable geometry, animation, and rendering so that a single loss on a synthesized image or simulation output can propagate gradients all the way back to material parameters, control inputs, or geometric design variables.
  • Hybrid neural + visual models: strategies for compositing learned neural network blocks (neural operators, residual networks, adaptation layers) within the DVC pipeline to handle difficult-to-model phenomena while retaining physically grounded gradient flow.

② Embed differentiable physics priors into geospatial foundation models

Extending this paperThe review argues that DVC's key advantage over pure deep learning is data efficiency through physics priors. It cites robotics and manufacturing as primary use cases, but the same principle generalizes: any domain where a forward physical model exists (diffusion, transport, gravity) can benefit from making that model differentiable and embedding it as a constraint in a learned system. Extending the paper: incorporate differentiable urban mobility or epidemic spreading models (analogues of differentiable physics simulators) as physics-constrained encoders within large-scale geospatial foundation models, reducing the labeled data they need while improving extrapolation.

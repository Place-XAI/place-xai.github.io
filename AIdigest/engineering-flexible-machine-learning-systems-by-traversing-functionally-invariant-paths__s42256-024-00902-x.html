Engineering flexible machine learning systems by traversing functionally invariant paths

Guruprasad Raghavan · Bahey Tharwat · Surya Narayanan Hari · Dhruvil Satani · Rex Liu · Matt Thomson Published 2024-10-03 DOI 10.1038/s42256-024-00902-x Source nature.com ↗
Core Conclusion

Standard neural network training yields a single fixed weight configuration, leaving no room for adaptation without catastrophic forgetting. This paper introduces Functionally Invariant Paths (FIPs) — a differential geometry framework that treats the neural network weight space as a Riemannian manifold and constructs geodesic paths along which a network's input–output behavior is preserved while secondary objectives (additional tasks, sparsification, adversarial robustness) are progressively achieved. Applied to BERT, ViT, DeIT, and CNNs with modest GPU resources, FIP matches or exceeds state-of-the-art performance on continual learning (91.2% on SplitCIFAR with ViT-B across 5 tasks), sparsification (80% weight removal with minimal accuracy loss on ImageNet1K), and adversarial ensembling (55.6% adversarial accuracy vs. 35% for independently trained ensembles).

Why It Matters

  • Unifies three hard ML problems under one mathematical framework. Continual learning, network sparsification, and adversarial robustness have traditionally been addressed by separate, architecture-specific heuristics; FIP subsumes all three via a single geometric objective.
  • Shifts the paradigm from single networks to path-connected sets. Rather than optimizing one weight configuration, FIP discovers an entire submanifold of isoperformance networks — a richer computational resource that can be queried for new objectives.
  • Scales to hundred-million-parameter transformers on academic hardware. ViT-B (86M) and BERT (110M) experiments run on an NVIDIA RTX2060/RTX3090, making the method accessible without industrial-scale clusters.
  • Mathematically grounds what was previously heuristic. Methods like LoRA, EWC, and OGD are local heuristics with no global optimality guarantee; FIP provides a metric-tensor construction defined at every point in weight space, enabling long-range exploration that beats LoRA on catastrophic forgetting across all tested rank settings.
  • Ships as open code under MIT license. Executable PyTorch implementation is publicly available at github.com/Guruprasad93/FlexibleMachineLearning, together with a full dataset list.

Core Methods

  • Riemannian metric tensor on weight space: the Jacobian J of the network output f(x; w) with respect to weights w is used to define a local metric tensor g = JTJ at every point in weight space. The metric measures how much a weight perturbation dw changes the network's functional output, providing a first-order (not second-order/Hessian) tool for discovering low-eigenvalue "flat" directions.
  • Spectral analysis of the metric: the eigenspectrum of g reveals a large number of near-zero eigenvalues (consistent with the Marchenko–Pastur law for overparameterized networks), defining a high-dimensional functionally invariant subspace where weight changes have negligible effect on network behavior.
  • FIP path-finding optimization: at each step, solve a constrained quadratic program (equation 4) that simultaneously minimizes output-space velocity (i.e., stays on the isoperformance manifold) and maximizes alignment with the gradient of a secondary objective L(x, w) — whether it be a new classification loss, a sparsity-inducing term, or an adversarial-diversity criterion.
  • Continual learning (CL) application: FIP extends a base network to new tasks (SplitCIFAR, BERT Yelp→IMDb) by traversing weight-space paths that preserve output on previous tasks while improving on new ones; outperforms LoRA (which causes complete forgetting of Task 1) and matches/exceeds EWC, GEM, RMN.
  • Network sparsification: secondary objective L = ‖w − project(w, p)‖² drives the FIP toward a p%-sparse submanifold; achieves 80% sparsity on DeIT-B on ImageNet1K with minimal accuracy loss and spans full sparsity ranges 0–90% for BERT on GLUE.
  • Adversarial ensemble generation: FIP without secondary objective (L = 0) generates path-connected diverse networks; a selection criterion based on output-space distance under image perturbation identifies ensemble members with high representation diversity and low coherence with the attack surrogate, yielding 55.6% ensemble adversarial accuracy vs. 35% baseline.
  • Non-Abelian composition of model operations: iterating FIP transformations (CL then compress, or compress then CL) produces distinct weight configurations but comparable functional performance, establishing non-commutativity (non-Abelian nature) of model-adaptation operations.

① Apply FIP-style continual learning to geospatial foundation model fine-tuning across urban contexts

Extending this paperThe paper demonstrates FIP on language (BERT on Yelp→IMDb) and vision (ViT on CIFAR sub-tasks), but the framework is architecture-agnostic. A direct extension would be to apply FIP to geospatial foundation models (e.g., models pre-trained on remote sensing imagery or GPS trajectories) that must be sequentially fine-tuned across heterogeneous urban contexts — e.g., a model first calibrated for a North American city then adapted to an Asian megacity without forgetting the original calibration. The FIP's Riemannian metric would preserve the model's prior spatial representations while navigating new domain statistics, addressing one of the core challenges in generalizable GeoAI: the spatial non-stationarity that makes standard fine-tuning catastrophically forget prior geographic knowledge.

The FIP framework would give my fine-tuning pipeline a mathematically principled continual-adaptation mechanism, letting me sequentially incorporate data from new regions (mobile phone records, POI data, social media) without retraining from scratch, preserving cross-city transferability while adapting to local spatial patterns.

② Use FIP sparsification to build lightweight, deployable spatial ML models for edge and low-resource settings

Extending this paperThe paper achieves 80% weight sparsification on DeIT-B (ImageNet1K) and up to 90% sparsity on BERT GLUE tasks with minimal accuracy loss, using the FIP secondary objective to drive weights toward zero while preserving output behavior. An extension would be to apply FIP sparsification to large spatial prediction models that must run in low-resource environments — e.g., ML models predicting healthcare facility demand on mobile devices in low-bandwidth regions, or geospatial ensemble models that need to be compressed for edge deployment in public health surveillance. The FIP's unique strength here is that it generates the entire sparsity curve in one pass rather than requiring separate prune-retrain cycles for each target sparsity.

③ Construct FIP-generated ensembles for spatially robust prediction under distributional shift

Extending this paperThe paper's adversarial ensemble (Section "FIP ensemble confers robustness against adversarial attack") samples functionally diverse networks along an FIP and averages their outputs to resist PGD attacks on CIFAR-10. The adversarial input is analogous to — though not identical to — distributional shift in spatial data: if a model is trained on one city's mobility patterns and deployed in another, the shifted distribution functions like a "spatial adversary." Extending the paper: formalize a spatial adversary that perturbs geographic covariates (population density, network topology, land-use mix) within realistic bounds, then use FIP ensembles to achieve robust spatial prediction. The low-coherence selection criterion the paper introduces (cosine distance between loss-function gradients) could be adapted to select spatially diverse network members.

FIP ensembles would yield a principled uncertainty band over spatial predictions — complementing my SHAP/explainability layer with a geometric diversity measure that quantifies model disagreement across the geographic feature space, directly improving the reliability of GeoAI-informed policy recommendations.

④ Extend FIP's non-Abelian composition to multi-scale spatial modeling pipelines

Extending this paperSection "Non-Abelian, iterative transformation of BERT" demonstrates that applying CL then sparsification (CLCo) versus sparsification then CL (CoCL) produces models with identical functional performance but distinct weight configurations, making the operations non-commutative. The paper uses this to generate a 300-network submanifold. An extension would formalize the group structure of FIP transformations — treating each geographic scale (city, district, neighborhood) or data modality (remote sensing, POI, mobile phone) as a distinct transformation — and study whether the non-Abelian structure of the resulting model group encodes useful hierarchical information about the spatial domain. The submanifold could be queried not just by perplexity but by spatial prediction error on held-out regions.

Added value to my researchMy multi-source data fusion pipeline (combining remote sensing, social media, GPS, and POI data) performs a sequence of modality-specific model updates — analogous to FIP transformations with different secondary objectives. Recognizing the non-Abelian structure of these updates would allow me to design an optimal adaptation schedule (which data modality to incorporate first), rather than treating modality order as an arbitrary implementation detail, potentially improving convergence and final spatial accuracy for health geography models.

⑤ Embed FIP's Riemannian weight-space metric as a physics-informed constraint in graph neural networks for spatial accessibility

Extending this paperThe FIP framework's key innovation is replacing ad hoc heuristics with a metric-tensor–based constraint derived from the model's Jacobian — a structure that encodes the geometry of the model's input–output map at every point in weight space. This is directly analogous to physics-informed neural networks that embed conservation laws as soft constraints. An extension of the paper: derive a FIP-inspired spatial conservation constraint for graph neural networks operating on urban spatial graphs — for instance, constraining the GNN to preserve network-accessibility distances (akin to a "spatial metric") while learning new urban covariates, implementing this as a Riemannian metric on the GNN's weight manifold built from spatial adjacency Jacobians.

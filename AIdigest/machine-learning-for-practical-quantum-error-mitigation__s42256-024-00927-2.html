Machine learning for practical quantum error mitigation

Haoran Liao · Derek S. Wang · Iskandar Sitdikov · Ciro Salcedo · Alireza Seif · Zlatko K. Minev Published 2024-11-22 DOI 10.1038/s42256-024-00927-2 Source nature.com ↗
Core Conclusion

Classical machine learning — specifically random forest regression — can match or surpass the accuracy of conventional zero-noise extrapolation (ZNE) for quantum error mitigation, while delivering at least a twofold reduction in runtime overhead. Experiments on IBM quantum computers with up to 100 qubits and 1,980 CNOT gates demonstrate a scalable "mimicry" strategy: train ML models to replicate costly traditional mitigation methods, cutting overall quantum resource use by 25% and runtime overhead by 50%, without sacrificing accuracy.

Core Methods

  • ML-QEM general framework: train a classical ML model on (circuit features + noisy expectation values) → ideal expectation values, then deploy at runtime without extra mitigation circuits, eliminating the overhead bottleneck of conventional QEM.
  • Feature encoding: circuit-level features include counts of each native gate type (binned angles for parameterized gates), Pauli observable in sparse representation, and optional device-specific noise parameters (gate errors, decoherence times, readout errors). GNN additionally encodes the circuit as an undirected acyclic graph with node/edge vectors.
  • Four ML models benchmarked: (1) Linear regression (OLS) — interpretable baseline; (2) Random forest (RF) — 100-tree ensemble with bootstrap aggregation, consistently best-performing; (3) Multilayer perceptron (MLP) — two dense layers (hidden size 64, ReLU activation, Adam optimizer); (4) Graph neural network (GNN) — two multi-head Transformer convolutional layers + ASAPooling + two dense layers (hidden size 128).
  • Reference method (ZNE): digital zero-noise extrapolation with gate folding on 2-qubit gates, noise factors {1, 3}, and linear extrapolation — used as performance and overhead benchmark throughout.
  • Scalable mimicry strategy: when ideal expectation values are classically intractable (large circuits), train ML to mimic ZNE-mitigated values from hardware rather than ideal noiseless values, enabling error mitigation at 100-qubit scale.
  • Evaluation protocol: interpolation (unseen circuit parameters within training depth range) and extrapolation (circuit depths exceeding training set) tested separately; L2 error norm used as performance metric; bootstrap resampling (1,000 samples) for 95% confidence intervals.

① Transfer the paper's "mimicry" paradigm to spatiotemporal surrogate modeling

Extending this paperThe paper's "mimicry" strategy — train ML to replicate an expensive reference method rather than learn ideal outputs directly — is a general-purpose surrogate modeling paradigm. In this paper, the reference is ZNE; the surrogate is RF. A direct extension would test whether this mimicry generalizes across different hardware noise profiles (e.g., train on ibm_algiers noise, deploy on ibm_brisbane), testing whether the surrogate captures transferable noise structure or merely device-specific artifacts. Additionally, the current GNN architecture encodes quantum circuits as acyclic graphs; extending to dynamic, recurrent graph structures could better capture temporal noise drift mentioned in the Discussion.

Added value to my researchThe mimicry principle maps directly onto my spatiotemporal modeling agenda: rather than training geospatial foundation models from scratch on expensive labeled data, I can train lightweight ML surrogates to mimic the predictions of computationally expensive simulation models (e.g., SIR epidemic dynamics or physics-informed graph networks for traffic flows), drastically reducing inference cost. This is analogous to how RF-ZNE mimics ZNE in this paper, and pairs naturally with my multi-source data fusion work where a surrogate can bridge between data-rich and data-sparse regions.

② Replace random forest with graph neural networks for spatially structured observables

Extending this paperThe paper finds RF consistently outperforms GNN for quantum error mitigation, partly because GNN faces higher computational cost and implementation difficulty. However, the paper itself acknowledges a promising unexplored direction: if GNN is trained on large datasets across different noise models with noise parameters set to the low-noise regime, it may extrapolate to near-noise-free values without training on ideal outcomes. Extending the paper: systematically explore this "noise-parameter interpolation" regime in GNN, varying the encoded coherence times and gate errors in the node features to learn a smooth noise-to-ideal mapping. This would turn the GNN's currently underperforming model into a physics-aware extrapolator.

Added value to my researchMy physics-informed graph learning research directly benefits from this GNN noise-parameter interpolation paradigm. In spatial prediction tasks, varying the "noise" level corresponds to varying data sparsity or observation quality (e.g., coarser resolution remote sensing, lower mobile phone coverage). If a GNN can learn to extrapolate from noisy to ideal quantum observations by treating noise level as a continuous node feature, the same architecture could predict fine-grained spatial patterns from coarse satellite imagery by treating resolution as a learnable graph node feature — a generalizable physics-aware GeoAI operator.

③ Adapt the paper's feature encoding scheme for explainable ML in spatial prediction

Extending this paperThe paper encodes quantum circuit features (gate counts, Pauli observable in sparse representation, device noise parameters) as input vectors that augment raw noisy expectation values. This circuit-level feature engineering is what enables the RF to outperform simpler approaches. An important extension the paper does not pursue: use the trained RF model's feature importances to identify which circuit properties (gate depth, qubit connectivity, noise type) most strongly predict mitigation benefit — essentially applying explainable ML to quantum circuits to guide circuit compilation toward noise-robust structures.

Added value to my researchThe paper's feature engineering philosophy — encode domain-specific structural features alongside raw observations as a composite input — is transferable to explainable spatial ML. In my GWR/MGWR work, spatial non-stationarity is often opaque. Borrowing the paper's approach: augment tabular spatial features with graph-structural embeddings of the road or social network topology (analogous to circuit topology features), then apply RF with SHAP analysis to identify which spatial structural features drive local regression coefficient variation. This would transform my MGWR from a statistical tool into an interpretable GeoAI pipeline.

④ Test ML-QEM's interpolation/extrapolation boundary as a guide for geospatial transferability

Extending this paperA central finding is that ML-QEM performs well in interpolation (unseen parameters within the training range) but degrades in extrapolation (circuit depths beyond training). This is a universal challenge in supervised learning that the paper addresses only partially — it shows that coherent error suppression (dynamical decoupling + randomized compiling) recovers extrapolation performance by converting coherent noise to incoherent. Extending the paper: systematically characterize the interpolation/extrapolation boundary as a function of training set diversity and circuit complexity, establishing rules for how many and what types of training circuits are needed for robust deployment — a transfer learning analysis for quantum error mitigation.

Added value to my researchThe interpolation-vs-extrapolation framework maps precisely onto spatial transfer learning for GeoAI: training on cities in one density regime (e.g., high-density Asian megacities) and deploying in another (low-density African cities) is the geospatial analog of out-of-distribution Trotter steps. The paper's finding that suppressing coherent noise enables extrapolation suggests that "removing irreducible local structure" (analogous to normalizing for city-specific spatial autocorrelation) enables transferable geospatial models — a concrete hypothesis for my fine-tuning of geospatial foundation models across spatial regimes.

⑤ Apply multi-source data fusion to quantum noise characterization via heterogeneous ML ensembles

Extending this paperThe paper's RF model uses a fixed feature set (gate counts, noise parameters, noisy expectation values), but quantum devices produce rich heterogeneous data: pulse shapes, real-time crosstalk measurements, readout histograms, and time-stamped noise drift records. The paper explicitly notes that "better encoding strategies that incorporate other information about the circuit and noise model, such as pulse shapes and output counts, may lead to even more accurate error mitigation." Extending the paper: build a heterogeneous ML ensemble that fuses tabular circuit features (for RF), raw pulse waveforms (for 1D CNN), and circuit graph topology (for GNN) via a late-fusion stacking layer — a multi-modal ML-QEM that mirrors multi-source data fusion in geospatial AI.

Added value to my researchMulti-source data fusion is a core theme of my research: fusing mobile phone records, social media, remote sensing, and POI data for urban analytics. The paper's challenge (heterogeneous quantum device data with temporal drift) is structurally identical to my challenge of fusing asynchronous, multi-resolution spatial data streams.

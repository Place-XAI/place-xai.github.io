A cell atlas foundation model for scalable search of similar human cells

Graham Heimberg · Tony Kuo · Daryle J. DePianto · Omar Salem · Tobias Heigl · Nathaniel Diamant · Gabriele Scalia · Tommaso Biancalani · Shannon J. Turley · Jason R. Rock · Héctor Corrada Bravo · Josh Kaminker · Jason A. Vander Heiden · Aviv Regev Published 2024-11-20 DOI 10.1038/s41586-024-08411-y Source nature.com ↗
Core Conclusion

SCimilarity is a metric-learning foundation model that embeds single-cell RNA-seq profiles into a unified 128-dimensional space, enabling rapid similarity search across a 23.4-million-cell, 412-study atlas. Applied to interstitial lung disease, it retrieved fibrosis-associated macrophage states across multiple fibrotic diseases and tissues in seconds, and — without any prior hypothesis about lung biology — identified a 3D hydrogel PBMC culture system as the top in vitro hit, which was experimentally validated. The model outperforms existing single-cell foundation models (scGPT, scFoundation) on cell querying while matching dedicated integration methods on batch correction.

Why It Matters

  • Scales cell search to tens of millions of profiles. Previous single-cell tools were designed for integration and annotation, not for rapid querying at atlas scale. SCimilarity searches 23.4 million cells in 0.05 s, whereas a conventional gene-signature scan of the same set takes nearly 3 hours.
  • Creates a universal, annotation-free cell similarity metric. The learned metric does not require curated gene lists, cell type labels, or fine-tuning on new data — enabling cross-study, cross-tissue, cross-disease comparison out of the box.
  • Bridges in vivo and in vitro biology computationally. By embedding both tissue and culture system data in the same space, SCimilarity can predict which laboratory models recapitulate specific disease cell states — a powerful hypothesis-generation engine that was experimentally confirmed.
  • Interpretability via Integrated Gradients. Unlike black-box models, SCimilarity attributes similarity scores to specific genes, recovering known and novel markers of the queried cell state without pre-specified gene panels.
  • Open and reproducible. Model weights, k-NN indices, embeddings and code are deposited at Zenodo and GitHub under permissive licenses, directly enabling reuse and extension.

Core Data — Sources & Volume

Training corpus
412 human sc/snRNA-seq studies comprising 23,381,150 cell profiles from 5,142 tissue samples. Covers 184 unique Tissue Ontology terms and 132 Disease Ontology terms. Sourced primarily from Gene Expression Omnibus (GEO) (334 studies downloaded programmatically) and CELLxGENE (additional 66 studies plus large consortia). All data restricted to 10x Genomics Chromium droplet-based sc/snRNA-seq to minimise platform confounding.
Training / test split
Training set: 56 studies, 7,886,247 cells with author-provided Cell Ontology annotations (203 terms). Test set: 15 held-out studies, 1,415,962 cells — entire studies withheld, not random cells.
Triplet sampling
50,000,000 semi-hard triplets sampled from the training corpus, weighted by study and cell type. Hard triplet mining used to focus on the most informative comparisons.
Query reference
A 23.4-million-cell approximate k-NN index (hnswlib) — searched in 0.05 s for top 10,000 nearest neighbours, or 2 s for exhaustive comparison of 2.5 million monocytes/macrophages.

Core Methods

  • Deep metric learning with combined triplet + reconstruction loss: a fully connected encoder (28,231 genes → 1,024 → 1,024 → 1,024 → 128 dimensions, unit-hypersphere normalized) is trained jointly with a triplet loss (to place similar cells near each other across studies) and an MSE reconstruction loss (to preserve intra-type variation). The weighting parameter β = 0.001 balances query sensitivity and integration performance.
  • Ontology-aware triplet sampling: cell triplets are constructed using Cell Ontology ancestor-descendant relationships to exclude ambiguous (ancestor/descendant) pairings and focus only on unambiguously similar or dissimilar cell pairs — enabling seamless cross-study training without manual annotation harmonization.
  • Approximate nearest-neighbour indexing (hnswlib): a pre-built k-NN index over 23.4 million embeddings enables queries in 0.05 s (top 10,000 cells) versus 2 h 46 min for exhaustive gene-signature scoring.
  • Integrated Gradients explainability: an extended Integrated Gradients method computes per-gene attribution scores by measuring how each gene's expression along an interpolation path from a background cell to a query cell affects the latent-space distance — identifying the most discriminating genes for any queried cell state without pre-defined signatures.
  • Cell type annotation via distance-weighted k-NN classification: query cells are annotated by retrieving the 50 nearest neighbours in a 7.9-million-cell annotated reference and tallying weighted votes — taking 0.02 s per cell.
  • Benchmarking against dedicated integration methods: SCimilarity's representations are compared to Harmony, scVI, Scanorama, and scArches using ontology-aware ASW, ARI, NMI, and graph connectivity metrics on four held-out integration tasks.

① Extend SCimilarity's metric-learning paradigm to geospatial place embeddings

Extending this paperSCimilarity's core insight is that a contrastive triplet loss over a biological ontology produces a universal similarity metric for cells — bypassing the need for domain-specific feature engineering. The paper's model is limited to the cell modality and the Cell Ontology hierarchy. A direct extension: apply the same triplet-loss framework to multi-source geospatial representations (POIs, demographics, mobility flows, remote sensing) using a Place Ontology (e.g., urban land-use hierarchy or spatial administrative hierarchy) as the source of positive/negative pairs — creating a universal "SCimilarity for places" that retrieves spatially and functionally similar neighborhoods across cities worldwide.

Added value to my researchThis directly advances my agenda of fine-tuning geospatial foundation models: a contrastive place encoder trained on multi-source urban data would produce transferable place embeddings for downstream tasks (health access prediction, telehealth gap detection, facility-location optimization) without requiring task-specific labeled data — mirroring how SCimilarity annotates unseen cells without retraining.

④ Apply SCimilarity's ontology-aware training strategy to spatial disease surveillance

Extending this paperSCimilarity avoids manual annotation harmonization by using the Cell Ontology's ancestor-descendant structure to automatically determine which cell pairs are unambiguously similar or dissimilar — a principled solution to label granularity mismatch across studies. This strategy has not been applied to epidemiological disease ontologies. A direct extension: use the ICD/SNOMED disease hierarchy as the training signal for a similarity model over spatial disease burden profiles (e.g., county-level multi-morbidity patterns from EHR or claims data), enabling cross-region retrieval of areas with similar disease landscapes without requiring harmonized diagnostic coding across health systems.

Added value to my researchThis directly addresses the data fragmentation problem in my health geography work: different cities and health systems use incompatible coding granularities for primary care and mental health diagnoses. An ontology-aware contrastive encoder for spatial health profiles would enable retrieval of comparable regions for benchmarking 2SFCA accessibility gaps — without forcing coarse label harmonization that discards clinically meaningful distinctions.

⑤ Scale SCimilarity's query-coherence quality control to spatiotemporal mobility data curation

Extending this paperSCimilarity includes a practical quality-control mechanism: it sub-clusters the user's query cells into 10 subsets, runs independent searches from each subcluster centroid, and reports the mean overlap in query results as a "query coherence" score. High coherence (e.g., 94.7% for the FM query) validates that the query population is homogeneous enough to yield meaningful, reproducible results. This self-assessing query framework is absent from most large-scale data mining tools. A direct extension of the paper: apply the same coherence diagnostic to multi-study meta-analyses — automatically flagging when a pooled cell population is too heterogeneous for a single centroid query, and suggesting optimal sub-population splits.

Added value to my researchThe coherence framework maps directly onto a key challenge in my multi-source data fusion work: when aggregating GPS trajectories or mobile phone records from multiple cities or time periods, it is unclear whether the pooled mobility signature is homogeneous enough to train a shared model. Adapting SCimilarity's sub-cluster overlap metric to spatiotemporal mobility embeddings would provide a principled "data fusion quality score" — identifying when urban-mobility datasets from different cities or seasons are too dissimilar to merge, and guiding city-stratified or season-stratified modeling decisions.

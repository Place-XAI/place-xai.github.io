Predicting the state of charge and health of batteries using data-driven machine learning

Man-Fai Ng · Jin Zhao · Qingyu Yan · Gareth J. Conduit · Zhi Wei Seh Published 2020-03-02 DOI 10.1038/s42256-020-0156-7 Source nature.com ↗
Core Conclusion

Data-driven machine learning outperforms both equivalent circuit models (ECMs) and physics-based models (PBMs) for predicting battery state of charge (SOC), state of health (SOH), and remaining useful life (RUL), achieving mean errors of approximately 4% across all three quantities. No single ML technique dominates: neural networks excel for data-rich SOC tasks, while Gaussian processes and random forests are preferred for data-sparse SOH/RUL due to built-in uncertainty quantification. The field's future lies in hybrid physics–ML models trained on high-throughput automated experimental data, enabling real-time, explainable battery management.

Core Methods

  • Systematic literature review: structured survey of ECMs, PBMs (P2D model, single-particle model, reduced-order models), and DDMs with a unified accuracy-vs-CPU-time framework (Fig. 2).
  • ML techniques reviewed: linear regression, random forest / tree ensembles, support-vector machines (SVMs), Gaussian process regression, convolutional neural networks (CNNs), recurrent neural networks / LSTMs, and hybrid physics–ML approaches.
  • Validation protocols assessed: hold-out (80/20 train–test split) and k-fold cross-validation; error metrics include RMSE, coefficient of determination (R²), and mean percentage error.
  • Accuracy benchmark synthesis: aggregates percentage errors across 26+ studies — 4.0% (SOC), 5.0% (SOH), 4.1% (RUL) — providing the first consolidated cross-paradigm benchmark.
  • High-throughput pipeline design: describes a fully integrated automated workflow from electrode synthesis (thin-film sputtering, robotic combinatorial), high-throughput X-ray characterization (XRD, XRF, in-situ XAS), automated battery fabrication, to parallel electrochemical testing (cyclic voltammetry, galvanostatic cycling, EIS).
  • Active learning loop (proposed): ML surrogate model identifies high-uncertainty conditions, targets new simulations or experiments, retrains iteratively — reducing the labeled data required for accurate SOH/RUL prediction.

① Replace ad-hoc model selection with an uncertainty-driven active learning loop

Extending this paperThe paper's Table 2 shows that practitioners choose ML algorithms case-by-case — neural networks for data-rich SOC, Gaussian processes for data-sparse SOH — without a principled selection or acquisition protocol. A direct extension: replace this fixed choice with an uncertainty-driven active learning framework that, at each battery cycle, queries whichever operating condition has the highest predictive uncertainty, acquires that experimental data, and retrains. This would automate the model-selection trade-off the paper describes qualitatively, and reduce labeled data requirements for accurate SOH/RUL prediction by an order of magnitude — realizing the paper's envisioned "active learning" loop concretely.

Added value to my researchThe same acquisition logic applies directly to my spatiotemporal kernel density estimation and spatial ML workflows: treat unlabeled geographic locations as "battery cycles," measure prediction uncertainty from my ensemble, and selectively acquire ground-truth measurements (field surveys, sensor deployments) at the most informative spatial units — turning static training sets into adaptive geospatial active-learning pipelines that reduce costly data collection.

Discovery Learning predicts battery cycle life from minimal experiments

Zhang, Jiawei · Zhang, Yifei · Yi, Baozhao · Ren, Yao · Jiao, Qi · Bai, Hanyu · Jiang, Weiran · Song, Ziyou Published 2026-02-04 DOI 10.1038/s41586-025-09951-7 Source nature.com ↗
Core Conclusion

Discovery Learning — a scientific machine learning framework uniting active learning, physics-guided learning, and zero-shot learning in a human-like reasoning loop — predicts the cycle life of entirely new lithium-ion battery designs from only the first 50 cycles of 51% of prototypes, achieving 7.2% test error while saving an estimated 98% of testing time and 95% of energy compared with conventional full-life cycling. Trained exclusively on public datasets of different cell chemistries, it generalises to industrial-grade large-format pouch cells without requiring lifetime labels for the target design — demonstrating that zero-shot transfer + physics-guided features can replace years of destructive testing.

Why It Matters

  • Breaks the prototyping bottleneck. Battery development has historically required full-lifetime cycling of every new design — a process taking months to years. Discovery Learning compresses this to the first 50 cycles of about half the prototypes, enabling rapid design iteration critical for EV and grid storage markets.
  • Generalises across cell chemistries without target-specific labels. By combining zero-shot learning with physics-guided features, the framework predicts cycle life of a new design class using only historical data from different chemistries — the first demonstration of this capability at industrial scale with large-format pouch cells.
  • Quantifies massive resource savings. Under conservative assumptions, 98% time savings and 95% energy savings per design evaluation translate to dramatic reductions in battery development cost — making faster sustainable-energy transitions economically feasible.
  • Introduces a new paradigm: discovery-inspired ML. Drawing on educational psychology's Discovery Learning theory, the paper proposes a general human-like reasoning loop for scientific machine learning, with implications beyond batteries for any complex physical system with expensive experimental feedback.
  • Releases open data and code. The 123-cell industrial dataset and full codebase are available via Zenodo and Code Ocean, enabling community benchmarking and replication.

Core Methods

  • Discovery Learning loop: a three-component reasoning system — an Interpreter (physics-guided feature extraction from early cycles), an Oracle (zero-shot lifetime predictor trained on historical designs), and a Learner (active-learning module that selects which prototypes to cycle further) — operated in both open-loop and closed-loop modes.
  • Physics-guided feature engineering: electrochemical and degradation-physics features extracted from the first 50 charge/discharge cycles, encoding capacity fade trajectories, internal resistance, differential capacity analysis, and other physically meaningful signals that transfer across cell chemistries.
  • Zero-shot learning: the model is trained exclusively on public datasets of cell designs different from the industrial target; no lifetime labels for the target design class are required, enabling pre-prototyping predictions.
  • Active learning query strategy: the Learner selects the most informative prototypes to physically cycle, reducing required measurements to 51% of the prototype set while maintaining prediction accuracy.
  • Validation: 7.2% test error (mean absolute percentage error) on industrial-grade 123-cell dataset; savings of 98% time and 95% energy estimated under conservative assumptions against conventional full-life testing.

② Replace fixed physics features with a learnable physics-informed graph encoder

Extending this paperThe paper's Interpreter relies on hand-crafted electrochemical features (capacity fade, differential capacity analysis, internal resistance trajectories) extracted from the first 50 cycles. While grounded in domain knowledge, these features encode degradation physics through fixed transformations decided a priori. A natural extension: replace the Interpreter with a physics-informed graph neural network that treats a battery cell as a graph of electrochemical processes (electrode surfaces, electrolyte interfaces, current collector contacts) and learns latent representations that respect thermodynamic and kinetic constraints as soft losses — potentially discovering non-obvious degradation signatures beyond hand-crafted features. The graph structure provides an inductive bias that could improve zero-shot transfer by encoding structural similarity across cell geometries.

Added value to my researchThis connects to my physics-informed graph learning agenda: designing GNN architectures where physical laws (conservation, symmetry, decay constraints) are embedded as architectural priors rather than regularisation losses. For urban analytics, the same approach encodes spatial interaction laws (gravity, Tobler's first law) into the graph operator — yielding spatially transferable predictions. The battery graph analogy is intellectually fruitful for designing interpretable spatial encoders.

③ Scale the active-learning Oracle to a multi-fidelity experimental design loop

Extending this paperThe paper's active Learner selects which prototypes to cycle under a single-fidelity query model — you either cycle a cell for 50 cycles or you don't. In practice, battery testing has multiple fidelity levels: impedance spectroscopy (minutes), short-cycle aging (days), and full-life cycling (months). Extending Discovery Learning to a multi-fidelity Bayesian active learning setting — where the Learner decides not only which cell to measure but at what fidelity level — would further shrink the experimental budget. Uncertainty estimates from each fidelity tier would feed the Oracle's prior, enabling dynamic cost-accuracy trade-offs across the design space.

Added value to my researchThis multi-fidelity active learning concept translates to spatial data collection: I routinely fuse satellite imagery (low-resolution, global coverage), mobile phone data (medium fidelity), and survey data (high fidelity, sparse). Casting multi-source spatial data fusion as a multi-fidelity active query problem would let me optimise data collection strategies for urban health models — selecting which areas to deploy expensive field surveys given cheap remote-sensing priors.

⑤ Add explainability to the Oracle: which physics features drive lifetime transfer?

Extending this paperThe paper reports 7.2% test error but does not explain which physics-guided features of the first 50 cycles drive successful cross-chemistry transfer. A SHAP or integrated-gradients attribution over the Oracle's inputs would reveal whether differential capacity analysis, capacity fade slope, or internal resistance changes dominate prediction — and whether this ranking shifts across chemistries. This explainability layer would both validate the physics and identify which measurements are most information-rich, potentially reducing the 50-cycle window to fewer cycles for specific feature classes. Such an explainability study would strengthen the science case that physics-guided features carry genuine mechanistic meaning rather than being correlational shortcuts.

Multimodal learning with next-token prediction for large multimodal models

Wang, Xinlong · Cui, Yufeng · Wang, Jinsheng · Zhang, Fan · Wang, Yueze · Zhang, Xiaosong · Luo, Zhengxiong · Sun, Quan · Li, Zhen · Wang, Yuqi · Yu, Qiying · Zhao, Yingli · Ao, Yulong · Min, Xuebin · Men, Chunlei · Wu, Boya · Zhao, Bo · Zhang, Bowen · Wang, Liangdong · Liu, Guang · He, Zheqi · Yang, Xi · Liu, Jingjing · Lin, Yonghua · Wang, Zhongyuan · Huang, Tiejun Published 2026-01-28 DOI 10.1038/s41586-025-10041-x Source nature.com ↗
Core Conclusion

Emu3, a family of multimodal models trained solely with next-token prediction, matches the performance of well-established task-specific models across both perception and generation — without diffusion or compositional architectures. It unifies text, image, and video under a single decoder-only Transformer, demonstrates scaling laws consistent across modalities, and extends naturally to robotic manipulation and interleaved vision–language generation. This establishes next-token prediction as a general-purpose foundation for unified multimodal intelligence.

★

Why It Matters

  • Unifies a fragmented field with one objective. Multimodal AI has relied on incompatible paradigms — diffusion models for generation, CLIP-based encoders for understanding. Emu3 collapses all into one next-token loss, producing a single model that does it all.
  • Demonstrates consistent cross-modal scaling laws. The paper empirically establishes that T2I, I2T, and T2V all follow power-law scaling (β = 0.55 data exponent), giving practitioners a principled basis to forecast performance before full-scale training.
  • Removes the encoder prerequisite. By matching CLIP-initialized architectures without any pretrained vision encoder, Emu3 challenges the belief that strong unimodal priors are necessary — lowering the barrier for domain-specific multimodal model development.
  • Extends to embodied AI without task-specific components. Treating robotic actions as tokens in the same sequence, Emu3 competes with specialized manipulation models on the CALVIN long-horizon benchmark — opening a route to generalist robotic agents.
  • Fully open-source ecosystem. Code (Apache-2.0), tokenizer, pretrained weights, and post-trained derivatives are all public, accelerating reproducible multimodal research.

Core Methods

  • Unified discrete tokenization: a custom VQ-based vision tokenizer (SBER-MoVQGAN-based, 32,768-entry codebook) encodes 512×512 images and 4×512×512 video clips into 4,096 discrete tokens each, with 4× temporal and 8×8 spatial compression — enabling images, videos, and text to share one vocabulary.
  • Decoder-only Transformer with next-token prediction: an 8.49B parameter Transformer (Llama-2-style) trained end-to-end on interleaved multimodal sequences using a single cross-entropy loss; no diffusion objective, no separate vision encoder.
  • Three-stage curriculum pretraining: Stage 1 — high learning rate, no dropout, short context (5,120); Stage 2 — dropout (0.1) for stability; Stage 3 — extended context (65,536) for video with context parallelism.
  • Quality fine-tuning (QFT) + Direct Preference Optimization (DPO): post-pretraining alignment via aesthetics-scored image data (HPSv2.1, MPS, LAION-Aesthetics) and human-preference triplets to improve generation fidelity and alignment.
  • Scaling law analysis: power-law model L(N, D) = E + A/N^α + B/D^β fitted across three tasks; shared data exponent β = 0.55 and task-specific model exponents (α = 0.25 for T2I/I2T, α = 0.35 for T2V), validated with R² ≥ 0.99.
  • Alternative token orders: fine-tuning the pretrained model for diagonal, block-raster, and spiral-in token orders demonstrates that raster-trained priors transfer, enabling zero-shot inpainting via spiral-in decoding.
  • Vision–language–action (VLA) extension: robotic actions tokenized with FAST tokenizer (1,024-vocab), interleaved with vision and language tokens; trained on CALVIN benchmark with a sliding two-frame window for online prediction.

⑤ Use Emu3's discrete token space to build a multi-task health accessibility foundation model

Extending this paperThe paper shows that multi-task learning within one next-token objective (simultaneous T2I, T2V, I2T, and VLA) outperforms single-task fine-tuning in several ablations and avoids task-bias through balanced loss weighting. An extension: define a health-access multi-task objective where tasks share the same discrete token backbone — (1) predicting travel-time accessibility scores from map tokens (regression as token prediction), (2) generating explanatory text from spatial statistics (I2T), and (3) recommending facility placements from demand maps (VLA-style action tokens). The paper already demonstrates that this multi-task format generalizes: adding a new "task" requires only new special tokens and fine-tuning data, not architectural change.

The discrete-token framing also makes results inherently interpretable: each predicted token corresponds to a quantized accessibility or placement decision, which can be linked back to spatial features using my existing explainable-ML pipeline. This is the most direct path to a reproducible, open GeoAI platform for health-facility access analysis.

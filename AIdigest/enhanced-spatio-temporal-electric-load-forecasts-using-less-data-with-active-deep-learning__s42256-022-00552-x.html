Enhanced spatio-temporal electric load forecasts using less data with active deep learning

Arsam Aryandoust · Anthony Patt · Stefan Pfenninger Published 2022-11-15 DOI 10.1038/s42256-022-00552-x Source nature.com ↗
Core Conclusion

Electric utilities can use active deep learning (ADL) to strategically select where and when to collect smart-meter data, achieving 26–81% more accurate load predictions while using 29–46% less data — and reducing physical smart-meter deployment by up to 24% — compared with conventional passive (random) data collection, at the price of 4–11 times more computation. The framework leverages an embedding uncertainty metric derived from a novel spatio-temporal embedding network to cluster candidate buildings and times in feature space, querying the most informative profiles first.

Year 2014. Candidate data pools are formed from 800 time stamps (for the 100-building dataset) and 200 time stamps (for the 400-building dataset), selected randomly from 2014.

Data budget

Core Methods

  • Spatio-temporal embedding network: a modular deep neural network combining ANN, CNN, RNN, and CRNN sub-modules — one branch per feature type (time stamp, building image histogram, meteorological conditions) — that maps the concatenated feature vector (520 dimensions) to a 24-hour, 15-minute-resolution load profile (96 values).
  • Embedding uncertainty metric: each sub-encoder maps candidate data points into a 100-dimensional latent space. The Laplacian-kernel distance of each embedded candidate to its K-means++ cluster centre is used as an informativeness proxy — the paper's key methodological contribution.
  • Batch ADL algorithm: iterative pool-based active learning where (1) candidates are embedded, (2) K-means++ clusters them, (3) one candidate per cluster is selected based on the embedding uncertainty (four variants: random, maximized, minimized, averaged), and (4) the queried labels are used to retrain the DL model. The removal rate δ ∈ {0, 1} controls whether queried points are kept in the pool (incremental learning, δ = 0) or removed (forced exploration / catastrophic forgetting, δ = 1).
  • Passive deep learning (PDL) baseline: identical network trained on uniformly random data samples from the same candidate pool; used as the primary benchmark. A random forest (RF) also serves as a non-DL reference.
  • Computational complexity reduction strategies: subsampling the candidate pool (up to 75%), reducing cluster count, and querying by spatial coordinate embedding only — achieving 2–3× speedup with minimal accuracy loss.
  • Evaluation types: spatial predictions (unseen buildings, seen times), temporal predictions (seen buildings, unseen times), and spatio-temporal predictions (unseen buildings and times) — testing generalization across all distribution shift combinations.

① Replace the paper's fixed K-means++ clustering with a graph-structured ADL policy

Extending this paperThe paper's ADL algorithm clusters candidate data points in a flat embedding space, ignoring the spatial topology of the distribution network. Buildings do not exist in isolation — they are connected by grid infrastructure whose graph structure creates physical dependencies in load. A direct extension: reformulate the ADL candidate-selection step on a spatial graph where nodes are buildings (or substations), edges are grid lines, and the query policy is learned by a graph neural network that propagates uncertainty across connected nodes. This would make informativeness estimates physically meaningful rather than purely geometrically derived, and should further reduce data demand in realistic utility settings where grid topology is known.

Added value to my researchThis graph-aware ADL extension maps directly onto my physics-informed graph learning agenda: encoding grid topology as a soft physical constraint in the GNN embedding means the learned query policy generalizes across grids of different sizes — a transferable GeoAI sensor-placement operator. The same framework applies to my urban analytics context (facility catchment areas as a graph), turning passive data collection for spatiotemporal models into a principled allocation problem.

② Extend the embedding-uncertainty metric to multi-task and multi-city transfer settings

Extending this paperThe paper trains and evaluates entirely within a single country (Switzerland) on one year of data. The embedding uncertainty metric is computed from one task's encoder — but the same encoder could, in principle, be pre-trained on buildings from many cities and then fine-tuned. An extension: apply the paper's ADL framework in a transfer-learning regime where a model pre-trained on densely instrumented cities (e.g., European capitals with high smart-meter penetration) is fine-tuned with active queries on a target city with sparse data. The embedding uncertainty, now reflecting cross-city feature diversity, would guide where to install the first meters in a new city — turning the paper's within-sample query strategy into a city-level deployment planner.

Added value to my researchTransfer-based ADL translates to my fine-tuning of geospatial foundation models: instead of random fine-tuning samples, use embedding uncertainty to select the most geographically diverse and informationally rich locations — exactly the "multi-source data fusion" challenge I face when adapting mobile-phone or remote-sensing models across cities. This also aligns with my interest in open science/FAIR: a pre-trained embedding released publicly lets other groups run ADL for sensor placement without re-training from scratch.

③ Scale the paper's framework from buildings to city-wide net-zero planning

Extending this paperThe paper operates at the single-building level with 100–400 buildings in Switzerland. Its predictions are the load profile of one building given its aerial image and local meteorology — a fine-grained but spatially limited scope. A natural extension: aggregate the building-level ADL predictions upward to produce district- and city-scale load maps, integrating rooftop solar generation estimates (also derivable from aerial imagery) to produce residual-load (net-load) maps for active distribution grids. This would connect the paper's per-building forecasts to the urban-scale energy-transition planning literature, testing whether active selection of measurement sites at building level also minimizes aggregate city-scale uncertainty.

④ Apply the paper's ADL query logic to equitable smart-meter deployment in underserved cities

Extending this paperThe paper demonstrates that ADL can reduce the number of smart meters needed while maintaining forecast accuracy, but it never asks where those meters should go in an equity-conscious way. Fig. 1 highlights that smart-meter adoption in Africa and Latin America is near zero — exactly the context where ADL's meter-saving properties are most valuable. An extension: couple the paper's embedding-uncertainty-based placement algorithm with an equity constraint (e.g., minimum coverage fraction per income quartile or neighbourhood type), producing a placement plan that is simultaneously informative for the grid model and fair across socio-economic groups. This would transform ADL from a pure ML efficiency tool into an infrastructure-equity instrument.

⑤ Harvest the paper's uncertainty estimates as explainability signals for ML-based urban energy policy

Extending this paperThe paper's embedding-uncertainty metric reflects which spatio-temporal contexts the model finds most surprising or poorly represented — a latent measure of model confidence across space and time. This signal is currently used only for data selection, then discarded. An extension: retain and map the embedding uncertainty surface after the final ADL iteration to produce an "uncertainty atlas" of the grid — identifying neighborhoods, building types, and seasonal windows where load predictions remain least reliable. This atlas could guide targeted audits, serve as an explainability layer for regulators, and inform where additional physical sensors have the highest marginal value even after ADL has converged.

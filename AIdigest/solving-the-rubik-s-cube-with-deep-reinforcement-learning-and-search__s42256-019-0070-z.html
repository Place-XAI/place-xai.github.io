Solving the Rubik's cube with deep reinforcement learning and search

Forest Agostinelli · Stephen McAleer · Alexander Shmakov · Pierre Baldi Published 2019-07-15 DOI 10.1038/s42256-019-0070-z Source nature.com ↗
Core Conclusion

DeepCubeA combines deep approximate value iteration with batch weighted A* search (BWAS) to solve 100% of all test Rubik's cube configurations — finding a shortest path 60.3% of the time — without any domain-specific knowledge. The same algorithm generalizes to six other combinatorial puzzles (15-puzzle through 48-puzzle, Lights Out, and Sokoban), using far less memory than pattern databases and running over 200× faster than classical solvers on the 24-puzzle, demonstrating that a single learned cost-to-go function can serve as a universal heuristic for large-state-space planning.

Why It Matters

  • Beats domain-engineered solvers without hand-crafted knowledge. Classical pattern-database (PDB) solvers require 182 GB of memory and hand-tuned group-theory heuristics; DeepCubeA replaces this with a single trained DNN, solving the same problems faster with dramatically less memory.
  • Demonstrates scalable generalization. A single algorithm architecture solves puzzles spanning state spaces from 1013 (15-puzzle) to 1062 (48-puzzle) and a planning domain (Sokoban) with irreversible actions — without re-engineering for each environment.
  • Solves the "sparse reward" problem without reward shaping. The reverse-scrambling training curriculum elegantly sidesteps the fundamental difficulty that random play almost never reaches the goal state in large combinatorial spaces.
  • Opens a path to general planning AI. Problems with large state spaces and few goal states — common in robotics, logistics, and molecular design — are natural targets for this approach, well beyond puzzle-solving.
  • Ships a publicly accessible solver. A web server at deepcube.igb.uci.edu and a Code Ocean compute capsule make the approach immediately reproducible and usable by the community.

All training states are synthetically generated by randomly scrambling the goal state up to K moves (K = 30 for Rubik's cube; K = 500–1,000 for other puzzles). No real-world dataset is used. The DNN saw 10 billion training examples over 1 million iterations (36 hours on two NVIDIA Titan V GPUs).

Core Methods

  • Deep Approximate Value Iteration (DAVI): trains a DNN to approximate the optimal cost-to-go function J*(s) using one-step Bellman updates (Eq. 1). A slowly updated target network θe provides stable training targets; θe is only updated when loss falls below threshold ε = 0.05.
  • DNN architecture: two fully connected hidden layers (5,000 and 1,000 units) followed by four residual blocks (each with two 1,000-unit layers), batch normalization, ReLU activations, and a single linear output unit. One-hot encoding of puzzle states (e.g., 324-dim for Rubik's cube). Trained with ADAM, batch size 10,000.
  • Batch Weighted A* Search (BWAS): uses the learned J(x) as heuristic h(x) in f(x) = λg(x) + h(x). Expands N lowest-cost nodes in parallel per iteration to maximize GPU throughput. Hyperparameters λ = 0.6, N = 10,000 for Rubik's cube; λ = 0.8, N = 20,000 for 15/24/35-puzzles.
  • Reverse curriculum training: each training state is generated by scrambling the goal state k ∈ [1, K] times, ensuring the reward signal propagates outward from the solved state without manual curriculum engineering.
  • Admissibility empirical analysis: the learned heuristic does not overestimate in 66.8% of states and overestimates by at most 1 in 97.4% of states (mean overestimation: 0.24 moves), providing a practical bound in lieu of a theoretical guarantee.
  • Distributed training: data generation across 6 GPUs, training on 2 GPUs, inference on 4 GPUs (NVIDIA Titan V / Pascal). Full Rubik's cube training completed in 36 hours with 10 billion examples seen.

① Replace the flat DNN heuristic with a GNN that generalizes across puzzle sizes

Extending this paperDeepCubeA trains a separate flat DNN for each puzzle family and size, requiring retraining from scratch when the environment changes (e.g., 24-puzzle to 35-puzzle). A direct extension is to replace the MLP with a graph neural network that encodes each puzzle state as a graph (tiles as nodes, adjacency as edges), enabling a single cost-to-go model to transfer across grid sizes without full retraining. This would test whether the learned heuristic is truly compositional or merely memorized for each state-space dimensionality, and could dramatically cut per-environment training cost.

Added value to my researchMy physics-informed graph learning work faces the same transferability challenge: GNN models trained on one city's road network need to generalize to another without full retraining. DeepCubeA's reverse-curriculum training — start from the solved state, explore outward — is directly analogous to training a spatial graph model on small, tractable subgraphs before scaling to city-wide networks. Combining reverse curriculum + GNN could yield a spatially transferable accessibility heuristic that avoids re-fitting 2SFCA service-area delineations for every new city or time period.

② Apply the reverse-curriculum principle to combinatorial facility-location optimization

Extending this paperDeepCubeA's pivotal insight — train on states generated in reverse from the goal — is a general strategy for any combinatorial search where forward random sampling almost never reaches the optimum. Extending it: treat facility-location problems (e.g., siting healthcare clinics across a city) as a puzzle where the "goal" is an optimal service distribution. Generate training states by randomly removing or displacing facilities from the optimal arrangement, then train a cost-to-go DNN on these reverse-scrambled configurations and deploy BWAS to find near-optimal placements. This would bring DeepCubeA's 100%-solve guarantee paradigm into spatial optimization.

Added value to my researchThis directly reframes my EV-charging and primary-care facility placement work: instead of solving a mixed-integer program for each city, a DeepCubeA-style value function trained on synthetically scrambled service distributions could let BWAS find near-optimal placements orders of magnitude faster. The reverse curriculum elegantly handles the sparse-reward problem that plagues RL for facility placement — most random arrangements score poorly, making reinforcement signals extremely thin, but starting from the optimum and scrambling outward guarantees dense, informative training data.

③ Use DeepCubeA's admissibility analysis as a template for calibrated uncertainty in spatial ML

Extending this paperThe paper's empirical admissibility analysis — showing the heuristic overestimates the cost to reach the goal in only 33.2% of states, and by at most 1 move in 97.4% of states — is a pragmatic substitute for the theoretical guarantee that A* requires. Extending it: model the overestimation distribution as a function of scramble depth (state difficulty) and use a conformal-prediction wrapper around the DNN to produce calibrated confidence intervals on move counts. This would let BWAS decide dynamically when to trust the heuristic and when to revert to a more conservative search strategy, yielding provable approximation guarantees in domains where admissibility is intractable to prove analytically.

Added value to my researchMy explainable ML work for spatial prediction suffers from the same calibration gap: point estimates without uncertainty bounds are hard for planners to act on. The DeepCubeA admissibility-analysis workflow — empirically characterize where and by how much the model overestimates, then bound the overestimation — is directly transferable as a conformal prediction framework for spatial ML (e.g., 2SFCA demand estimates, travel-time accessibility scores), giving policymakers probabilistic assurance rather than opaque point predictions.

④ Extend BWAS to time-varying routing problems for dynamic spatiotemporal accessibility

Extending this paperAll seven environments in this paper are static — the state space and transition costs are fixed. A natural extension is to benchmark DeepCubeA on time-varying combinatorial problems (e.g., dynamic vehicle routing or time-dependent shortest paths on congested road networks) where the optimal cost-to-go changes with time. This would test whether the value function can be conditioned on time as an additional input dimension and whether BWAS remains competitive under non-stationary heuristics — a significant open question for real-world planning applications.

Added value to my researchSpatiotemporal accessibility — how primary-care or telehealth catchment areas expand and contract by time of day, or how broadband availability shifts digital-health access — is a core theme in my research. A time-conditioned DeepCubeA value function trained on time-sliced GPS trajectory or mobile-phone record data could yield a dynamic 2SFCA solver that automatically adapts service-area delineations to time-varying travel times, replacing the static radius assumption with a learned, data-driven boundary updated per time slice.

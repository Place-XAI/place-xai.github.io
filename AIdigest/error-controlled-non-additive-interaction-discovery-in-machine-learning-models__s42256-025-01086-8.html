Error-controlled non-additive interaction discovery in machine learning models

Winston Chen · Yifan Jiang · William Stafford Noble · Yang Young Lu Published 2025-09-22 DOI 10.1038/s42256-025-01086-8 Source nature.com ↗
Core Conclusion

Diamond is a method for trustworthy feature interaction discovery in machine learning models that uniquely integrates the model-X knockoffs framework to control the false discovery rate (FDR). By introducing a non-additivity distillation procedure that strips out marginal and bias effects from off-the-shelf importance measures, Diamond isolates genuine synergistic interactions and maintains rigorous error control — a capability no prior approach had demonstrated. Validated across simulated datasets, a diabetes cohort (n=442), Drosophila enhancer genomics (n=7,809), and NHANES mortality data (n=14,407), Diamond enables reliable, data-driven scientific hypothesis generation across DNNs, transformers, tree-based models, and factorization machines.

  • Closes a fundamental gap in XAI. Existing methods rank feature pairs by joint importance, not by true non-additive synergy — and none control the proportion of spurious discoveries. Diamond is the first to deliver error-controlled interaction discovery in generic ML models.
  • Enables trustworthy scientific hypothesis generation. Without FDR control, a ranked list of feature interactions cannot be systematically cut off at a confidence threshold. Diamond replaces arbitrary thresholding with a principled statistical guarantee.
  • Robustness across model families. Diamond works on MLPs, transformers (FT-Transformer), KANs, XGBoost, LightGBM, and factorization machines — making it plug-and-play for any ML pipeline without architectural changes.

  • Fully open and reproducible. Code and all evaluation datasets are available on GitHub and archived via Zenodo, lowering the barrier to adoption across scientific domains.

Core Methods

  • Model-X knockoffs framework: generate dummy features that mimic the empirical dependence structure of the original features while being conditionally independent of the response, enabling FDR estimation without requiring p-values from the ML model.
  • Non-additivity distillation: decompose reported pairwise interaction importance eij into label-dependent marginal effects, label-independent feature biases, and residual non-additive interaction effects sij via a generalized additive model regression (pyGAM). Only sij enters FDR estimation.
  • Knockoff statistics and FDR threshold: a coin-flip antisymmetric statistic Wj compares original-feature vs. knockoff-feature interaction importance; a heterogeneous threshold T bounds the FDR across original-only, original–knockoff, and knockoff–knockoff interaction pairs at user-specified level q.
  • Pairwise-coupling plug-in layer: a knockoff-tailored architectural addition for DNN models (MLP, FT-Transformer) that pairs each original feature with its knockoff counterpart to maximize statistical power while preserving FDR control.
  • Interaction importance measures: Expected Hessian and Integrated Hessian (model-agnostic, via Path-Explain library), TreeSHAP for tree-based models, and learned FM coefficients for factorization machines.
  • Knockoff generation strategies: KnockoffsDiagnostics (primary, non-Gaussian), KnockoffGAN, Deep knockoffs, and VAE knockoffs tested for robustness; independently permuted features used as invalid-knockoff worst-case stress test.
  • Evaluation: empirical FDR (false-discovery proportion), statistical power, and AUROC over 20 independent repetitions per setting with 95% confidence intervals; fivefold cross-validation for real-data model selection.

② Replace heuristic predictor-pair selection in GWR/MGWR with Diamond's non-additivity distillation

Extending this paperThe paper shows that naive pairwise feature importance scores (SHAP interactions, Hessian-based measures) fail to control the FDR because marginal effects and feature biases contaminate the interaction signal. An identical problem arises in geographically weighted regression (GWR and MGWR) where practitioners manually select predictor-interaction terms based on unstandardized importance rankings or domain intuition. Extending Diamond to spatially indexed tabular data — where features include POI densities, demographic rates, or built-environment metrics — would introduce the first principled, error-controlled spatial predictor-interaction discovery workflow, decoupling the choice of interaction terms from arbitrary thresholding.

Added value to my researchIn my MGWR workflow I select predictor interactions (e.g., land-use mix × transit density for job-housing accessibility) based on domain knowledge or univariate importance, not on FDR-controlled evidence. Diamond's non-additivity distillation, applied as a post-hoc layer on top of any spatial prediction model, would let me systematically and defensibly identify which spatial feature-pairs require interaction terms — replacing intuition with statistical discipline across heterogeneous city-scale regression surfaces.

④ Use Diamond's FDR-controlled interaction list as a constraint graph for spatial causal structure learning

Extending this paperThe paper acknowledges that Diamond cannot distinguish direct interactions from transitive ones and identifies causal inference as a promising future direction. A practical extension: use Diamond's FDR-controlled non-additive interaction list as a constraint graph to initialize a causal structure learning algorithm — only feature pairs that pass Diamond's FDR threshold are candidates for directed edges. This hybrid pipeline (FDR-controlled interaction discovery then causal graph pruning) would dramatically reduce the exponential search space of causal learning while maintaining statistical rigor on the initial candidate edge set, addressing a core scalability bottleneck of high-dimensional causal learning.

⑤ Pair Diamond's error control with cross-cohort generalization tests for robust spatial biomarker discovery

Extending this paperDiamond ensures low false-discovery rates within a single training dataset, but the paper does not evaluate whether discovered interactions replicate across cohorts or geographic strata. The next extension: pair Diamond's FDR-controlled interaction list with a causal-modelling generalization test — for each discovered pair, assess whether its non-additive effect is stable across sub-populations, geographic regions, or temporal windows. This would extend the paper's statistical guarantees from within-sample FDR to cross-cohort reproducibility, addressing a core translational gap in biomarker research and making Diamond's output actionable for clinical or public-health policy.

Added value to my researchMy reproducibility and FAIR data science work focuses on ensuring that spatial ML findings replicate across cities and data vintages.

Learning conformational flexibility of immune receptors

Long-Chen Shen · Dong-Jun Yu · Jiangning Song Published 2025-10-16 DOI 10.1038/s42256-025-01133-4 Source nature.com ↗
Core Conclusion

Proteins are not static: despite the success of deep learning models in predicting fixed conformations, immune receptors such as antibodies and T-cell receptors exhibit critical dynamic flexibility at their CDR loops. ITsFlexible is a graph-based deep learning tool trained on a novel ALL-conformations dataset — assembled from the Structural Antibody Database (SAbDab) and the Structural T-Cell Receptor Database (STCRDab) — to classify the conformational flexibility of CDR loop motifs, and its predictions are validated with cryo-EM evidence.

  • Fills the static-structure gap in structural biology AI. Dominant tools like AlphaFold2 and AlphaFold3 predict single lowest-energy conformations; ITsFlexible adds a complementary flexibility classifier, acknowledging that CDR loops sample multiple conformational states critical for antigen recognition.
  • New curated dataset for the community. The ALL-conformations dataset is the first purpose-built resource capturing experimentally observed CDR loop conformational diversity from SAbDab and STCRDab, enabling future dynamic-structure ML research.

Core Methods

  • Graph-based deep learning (ITsFlexible): CDR loop structures are encoded as graphs — nodes represent residues/atoms, edges encode spatial/sequential relationships — and a graph neural network classifies the dynamic flexibility of each loop motif.
  • ALL-conformations dataset construction: multiple experimentally captured conformations for the same CDR loop are aggregated from SAbDab and STCRDab, creating a diversity-aware training set that explicitly represents flexibility rather than a single minimum-energy structure.
  • Flexibility classification: the task is formulated as a classification problem (flexible vs. rigid, or a multi-class spectrum) based on the structural heterogeneity observed across experimentally solved conformers.
  • Validation via cryo-EM: predicted flexibility classifications are cross-checked against independently determined cryo-EM structures (Fig. 1c), providing experimental ground truth beyond re-using the training database.
  • Workflow integration: the end-to-end pipeline (Fig. 1b) takes input CDR loop sequences/coordinates and outputs flexibility class predictions, designed for integration into antibody engineering workflows.

① Transfer ITsFlexible's graph encoding to spatial network classification problems

Extending this paperITsFlexible encodes a CDR loop as a graph over residue nodes with structural edges, then classifies the dynamic regime of the whole graph. The paper is silent on whether the learned graph representations generalize to CDR loops from different species or synthetic antibody libraries. A direct extension: apply the same multi-conformation graph encoding to ensemble MD trajectories (e.g., from MISATO) to test whether the classifier can predict flexibility from sequence alone without any experimental conformer, vastly expanding its applicability.

Added value to my researchThe conceptual machinery — encoding a structured object as a graph, then classifying its dynamic regime — maps directly onto my physics-informed graph learning agenda. Spatiotemporal networks (road networks under disruption, health-care access graphs under demand shocks) also have "dynamic regimes" that differ structurally from static topology. ITsFlexible's approach of training on observed multi-state ensembles rather than a single snapshot inspires an analogous training strategy for spatial accessibility graphs where temporal slices represent dynamic states.

② Add contrastive pre-training to the ALL-conformations dataset for richer flexibility representations

Extending this paperITsFlexible is trained in a supervised classification setting on the ALL-conformations dataset. A natural extension: apply molecular contrastive learning over pairs of conformers of the same CDR loop (a positive pair) versus conformers of different loops (negative pairs), building a richer self-supervised representation before fine-tuning for the flexibility classification head. This would reduce dependence on labeled flexibility classes and likely improve generalization to underrepresented loop geometries.

Added value to my researchThe contrastive pre-training idea translates to my spatiotemporal domain: treating two temporal snapshots of the same spatial unit (e.g., a census tract at two time points) as positive pairs and cross-geography pairs as negatives to learn spatiotemporal representations for downstream tasks like health outcome prediction or accessibility change detection — a geospatial contrastive pre-training framework.

④ Extend the classification to a generative flexibility-conditioned design loop

Extending this paperITsFlexible is a discriminative classifier: given a loop, predict its flexibility class. The natural inversion: a generative model conditioned on a target flexibility class that produces novel CDR loop sequences guaranteed to adopt that dynamic regime — closing the loop between classification and design. This would build on the paper's ALL-conformations dataset and graph representation, combining them with a diffusion or flow-based generative backbone to enable flexibility-by-design antibody engineering.

Extending this paperExisting GNN-based protein model refinement (e.g., rapid protein model refinement by deep learning) refines a single structure toward its lowest-energy conformation. Extending this paper: use ITsFlexible's flexibility predictions as a soft prior during GNN refinement — loops classified as flexible are allowed to explore wider conformational space during the refinement pass, while rigid loops are penalized for large deviations. This would unify flexibility classification (ITsFlexible) with structural refinement in a single joint framework, producing not just a refined single structure but a weighted ensemble of conformers.

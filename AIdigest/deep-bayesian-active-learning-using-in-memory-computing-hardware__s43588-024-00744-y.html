Deep Bayesian active learning using in-memory computing hardware

Yudeng Lin · Bin Gao · Jianshi Tang · Qingtian Zhang · He Qian · Huaqiang Wu Published 2024-12-23 DOI 10.1038/s43588-024-00744-y Source nature.com ↗
Core Conclusion

A memristor-based computation-in-memory (CIM) system can natively execute deep Bayesian active learning (DBAL) by exploiting the intrinsic stochasticity of memristor read and write operations to generate Gaussian random numbers in situ — eliminating the von Neumann bottleneck of conventional hardware. A four-layer memristor Bayesian deep neural network trained via the proposed memristor stochastic gradient Langevin dynamics (mSGLD) method achieved a 44% speed boost and 153× energy savings versus cutting-edge CMOS implementations, successfully demonstrated on a robot pouring skill-learning task with as few as 64 labeled samples.

Core Methods

  • Memristor stochastic CIM (ESCIM): crossbar array of 1T1R memristor cells performs in-memory vector–matrix multiplication (VMM) via Ohm's law + Kirchhoff's current law in a single parallel read, eliminating memory–processor data transfer.
  • In situ Gaussian random number generation: the intrinsic read noise of memristors follows a double-exponential (Laplace) distribution; summing currents from three devices invokes the Lindeberg–Feller central limit theorem to produce a Gaussian weight — no external RNG circuit needed.
  • mSGLD (memristor Stochastic Gradient Langevin Dynamics): sign-backpropagation maps gradient sign to set/reset device operations; conductance modulation stochasticity provides the Gaussian noise injection required by SGLD; a smooth transition method gradually reduces the weight update ratio to shift from gradient-dominated to noise-dominated phases.
  • Deep Bayesian active learning loop: (i) ex situ pre-training on digital computer using a small labeled set; (ii) deploy weights to ESCIM; (iii) memristor BDNN predicts uncertainty of 10K unlabeled candidates via repeated stochastic forward passes; (iv) straddle algorithm selects highest-uncertainty sample; (v) oracle labels it; (vi) in situ mSGLD retraining; repeat.
  • Uncertainty quantification via straddle algorithm: Uy = −|μy| + 1.96σy, computed from M repeated stochastic forward passes over memristor crossbar; no dropout required — device variability provides natural weight sampling.
  • Bayesian loss function (KL + log-likelihood): KL divergence between approximate posterior weight distribution and Gaussian prior, plus expected log-likelihood, balanced by hyperparameter β; gradients computed on a digital computer via PyTorch automatic differentiation.
  • Energy and latency benchmarking: ESCIM system measured against a CMOS GPU baseline; breakdown shows ADC dominates energy cost (95.2% at 130 nm), pointing to the key optimization target.

① Extend mSGLD to fully analogue reinforcement learning for adaptive spatial decision-making

Extending this paperThe paper demonstrates mSGLD for supervised Bayesian active learning — the oracle provides a binary label per query. A direct extension is to replace the external oracle with a reward signal from the environment, turning the framework into an analogue RL loop. The ESCIM's stochastic weights already provide natural exploration noise (an analogue of ε-greedy or policy entropy), and the mSGLD update rule generalizes naturally to policy gradients. The paper's smooth-transition trick (decreasing update ratio) could govern exploitation annealing. This would make the system the first fully analogue Bayesian RL chip, removing the digital computer from the inner loop entirely and completing the vision sketched in the Discussion section.

Added value to my researchSpatiotemporal facility-location problems — EV charging station placement, healthcare facility siting — are sequential decision problems where labeling costs are real (installing or moving facilities is expensive). An analogue Bayesian RL agent running on neuromorphic hardware could explore candidate locations on-device with uncertainty-aware sample efficiency, directly relevant to my equity-efficiency optimization agenda where repeated simulation evaluations are the labeling bottleneck.

② Combine DBAL uncertainty sampling with random-sketch compression for geospatial foundation-model fine-tuning

Extending this paperThe paper's DBAL loop selects the single highest-uncertainty sample per round to query for a label — a conservative batch-size-1 policy that works well on a small memristor BDNN (11×50×50×2) but does not scale to large-parameter foundation models. A natural extension: apply random-sketch dimensionality reduction (compressing the gradient or weight space) before deploying the DBAL loop on a larger memristor array, so the stochastic CIM system can handle BDNN layers with orders-of-magnitude more parameters without blowing the memory budget. The sketch preserves the Gaussian structure exploited by mSGLD, so uncertainty capture remains valid.

Added value to my researchFine-tuning geospatial foundation models (e.g., SatMAE, GeoCLIP) for localized health-geography tasks faces exactly this label-scarcity / compute-budget tension. A sketch-compressed DBAL strategy would let me identify which satellite tiles or geographic subregions to label first for a downstream accessibility prediction task, reducing annotation cost while maximizing model calibration — directly coupling my GeoAI agenda to hardware-efficient active learning.

③ Port the paper's uncertainty-driven query strategy to interatomic-potential-style spatiotemporal prediction

Extending this paperThe paper's straddle uncertainty metric — Uy = −|μy| + 1.96σy — is domain-agnostic: it selects the unlabeled point whose predicted distribution most straddles the decision boundary. A direct paper extension is to replace the robot pouring task with a regression-style active learning scenario (e.g., predicting a continuous outcome) by swapping the straddle criterion for query-by-committee variance or max-entropy, and testing whether the mSGLD training still converges on the same 4K chip. This would validate the framework's generality beyond binary classification and is the natural next experiment the Discussion section calls for.

④ Replace the digital-computer gradient backend with an on-chip synaptic learning rule toward a fully integrated system

Extending this paperThe paper is explicit that gradient computation (backpropagation) still runs on a tethered digital computer, which is the system's remaining von-Neumann bottleneck. The Discussion calls this out as a key limitation. A direct extension: exploit recent findings that standard silicon transistors can mimic synaptic and neural dynamics (short-term plasticity, spike-timing-dependent plasticity) to implement local learning rules — such as contrastive Hebbian learning or equilibrium propagation — on-chip, eliminating the digital compute tile entirely. The mSGLD sign-gradient rule is already a step in this direction (it only needs the sign, not the full gradient magnitude), making the transition architecturally feasible.

⑤ Scale mSGLD to nanomaterial discovery by coupling in-memory uncertainty sampling with physics-infused graph neural networks

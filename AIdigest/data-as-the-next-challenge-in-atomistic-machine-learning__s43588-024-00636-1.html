Data as the next challenge in atomistic machine learning

Chiheb Ben Mahmoud · John L. A. Gardner · Volker L. Deringer Published 2024-06-12 DOI 10.1038/s43588-024-00636-1 Source nature.com ↗
Core Conclusion

As atomistic ML models become mainstream tools in molecular and materials research, the data used to train them are as important as model architecture itself. The field must urgently shift from single-purpose, researcher-curated datasets toward large, openly shared, FAIR-compliant datasets that support general-purpose and foundation models — and from there toward dataset and model distillation strategies that convert expensive foundation models into efficient, task-specific tools.

Core Methods

  • Supervised atomistic ML workflow taxonomy: the paper formalizes the two-component dataset (data locations xi and labels yi) and maps three strategies for generating xi — database mining, MD-based sampling, and de novo exploration.
  • Dataset classification framework: single-purpose (one property type, e.g., potential-energy surface only) → multi-purpose (multiple properties on shared structures) → general-purpose / foundation datasets (broad chemical and property scope, millions of configurations).
  • Dataset distillation: selecting compact, task-specific subsets from large foundation datasets so that smaller specialized models can be trained efficiently without restarting data generation.
  • Model distillation (teacher–student): compressing large, expensive general-purpose models into smaller, faster specialized models via knowledge distillation — analogized to techniques pioneered in large-language-model research.
  • Synthetic pre-training + fine-tuning: pre-training neural-network potentials on synthetically generated data (no real QM labels needed at this stage) followed by fine-tuning on genuine DFT calculations, reducing real-data demand.
  • FAIR data principles as a methodological prescription: findability, accessibility, interoperability, reusability applied to atomistic datasets — with version control and standardized APIs as key implementation requirements.
